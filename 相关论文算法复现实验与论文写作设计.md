## 1. 和当前论文主线的关系

当前论文主张已经收敛为：

1. 本文主模型 `full_dqn_gated_multitask_cnn_ppo` 不是统计显著的市场超额收益证明。
2. 主要贡献是把连续权重生成和调仓时机控制分离，并在统一 next-open、成本、约束、HPO 与统计检验框架下评估。
3. 当前结果支持“成本控制 / 换手控制”强于“显著收益提升”的解释。

新增两个相关论文复现 baseline 后，论文主线不应改成“复现并击败原论文”。正确定位是：

```text
新增 baseline 是 closest hybrid related-work baselines。
它们用于检验：在同一 ETF/LOF 数据、同一 next-open 执行、同一成本和约束下，本文的 binary rebalance gate 是否比相近 PPO-DQN hierarchy 或 DQN signal + optimizer 路线更适合成本敏感资产配置。
```

论文中必须保持三层结果区分：

1. 原论文报告结果：只在 Related Work / Background 中解释，不能进入本项目主表。
2. 平台内 clean-room reimplementation 结果：可进入主表，但必须标记 `platform_adapted`。
3. 本文主模型结果：继续作为 proposed model，与新增 closest-hybrid baselines 做统一比较。

## 2. 新增 baseline 身份

正式可排名条目固定为六个：

```text
ppo_dqn_hierarchical_reimplementation
hybrid_dqn_optimizer_equal_weight
hybrid_dqn_optimizer_markowitz_mean_variance
hybrid_dqn_optimizer_minimum_variance
hybrid_dqn_optimizer_sharpe_maximization
hybrid_dqn_optimizer_risk_parity
```

`hybrid_dqn_optimizer_reimplementation` 只作为算法族 alias，不得作为 HPO budget 单元，不得作为 rankable `paper_model_id` 行。

### 2.1 RP1 论文算法解释口径

RP1 是 PPO-DQN hierarchical / hybrid 方向的近邻工作。官方页面摘要显示，该文将 PPO 的策略更新稳定性与 DQN 的价值评估能力结合，用于动态资产配置；原实验使用中国 A 股股票、滚动窗口和 PPO/DQN 联合训练设定。

本平台复现版本必须写成：

```text
clean-room platform-adapted reimplementation of a PPO-DQN hierarchical portfolio-allocation idea
```

不能写成：

```text
official replication
exact replication
same experiment as Liang (2025)
```

平台适配点：

- 原论文 PPO-DQN 细节并未完全冻结到可直接照搬的统一执行动作协议。
- 本平台首版固定 `dqn_role=high_level_action_selector`。
- DQN 在五个 high-level hierarchy actions 中选择：
  - `use_ppo_candidate`
  - `blend_current_ppo_25`
  - `blend_current_ppo_50`
  - `blend_current_ppo_75`
  - `fallback_equal_weight`
- 这不是本文主模型的 binary hold/rebalance gate。
- blend 动作必须记录 `platform_adapted_surrogate=true`。

### 2.2 RP2 论文算法解释口径

RP2 是 DQN signal + portfolio optimizer 方向的近邻工作。官方页面摘要显示，该文把 DQN 的离散交易信号与传统组合优化器分离：DQN 选择候选资产，优化器计算权重；比较的优化器包括 1/N、Markowitz Mean-Variance、Global Minimum Variance、Risk Parity 和 Sharpe Ratio Maximization。

本平台复现版本必须写成：

```text
clean-room platform-adapted reimplementation of a DQN signal plus portfolio-optimization framework
```

平台适配点：

- 五个 optimizer 必须是五个独立 child model。
- 每个 child 独立训练 DQN signal，独立 validation selection，独立 final test。
- 不能共享一个 DQN 后对五个 optimizer 做 final test 并声称同预算正式比较。
- DQN reward 是组合级真实 reward，必须写：
  - `portfolio_level_reward_shared=true`
  - `counterfactual_asset_reward=false`
- `factorized_q=true` 和 `platform_adapted_approximation=true` 必须进入 metadata。

## 3. 实验问题

新增实验块建议命名为：

```text
P9 closest hybrid related-work reimplementation
```

核心问题：

1. `RQ-RP1`: 在统一平台协议下，PPO-DQN high-level hierarchy selector 是否优于本文的 binary rebalance gate？
2. `RQ-RP2`: DQN signal + portfolio optimizer 的五个 child 中，哪类 optimizer 在 ETF/LOF next-open 协议下更稳健？
3. `RQ-RP3`: 相比 `dqn_template_native`，DQN 用作 asset signal 或 high-level selector 是否比直接模板选择更合理？
4. `RQ-RP4`: 若新增近邻 baseline 接近或超过本文主模型，本文贡献应改写为“统一协议下的结构比较”，而不是“主模型绝对优势”。

## 5. 指标与统计

### 5.1 主指标

继续沿用当前论文口径：

- primary ranking metric: post-cost cumulative return。
- cost discipline:
  - average turnover
  - total transaction cost
- validation selection:
  - `validation_penalized_sharpe`
  - canonical key: `validation_sharpe_minus_drawdown_turnover_penalty`

若加入风险调整指标，必须从 final test daily returns 重新计算：

- Sharpe
- Sortino
- Calmar
- maximum drawdown
- VaR
- CVaR

不得假设当前 `paper_main_comparison.csv` 已经天然包含所有风险指标。

### 5.2 必做 paired tests

新增正式 paired statistics 至少覆盖：

```text
full_dqn_gated_multitask_cnn_ppo vs ppo_dqn_hierarchical_reimplementation
full_dqn_gated_multitask_cnn_ppo vs hybrid_dqn_optimizer_equal_weight
full_dqn_gated_multitask_cnn_ppo vs hybrid_dqn_optimizer_markowitz_mean_variance
full_dqn_gated_multitask_cnn_ppo vs hybrid_dqn_optimizer_minimum_variance
full_dqn_gated_multitask_cnn_ppo vs hybrid_dqn_optimizer_sharpe_maximization
full_dqn_gated_multitask_cnn_ppo vs hybrid_dqn_optimizer_risk_parity
ppo_dqn_hierarchical_reimplementation vs ppo_native
hybrid_dqn_optimizer_sharpe_maximization vs dqn_template_native
```

统计解释：

- 所有统计都用 paired out-of-sample daily net returns。
- deterministic baselines 只有可匹配 seed/fold 时才做 paired tests。
- Holm correction 后不显著时，论文不能写成显著优于。
- 若 P9 新 baseline 优于主模型但不显著，应写“competitive / comparable”，不能写“dominates”。

## 8. 论文写作修改设计

### 8.1 Abstract

不建议在 Abstract 中加入新增 baseline 的具体数值，除非 P9 正式结果已经完成并通过复核。

若 P9 完成后支持当前主张，可加一句：

```text
Additional clean-room reimplementations of two closely related PPO-DQN and DQN-optimizer frameworks show that the proposed binary rebalancing gate remains competitive under the same execution and cost protocol.
```

若 P9 中新增 baseline 超过主模型，应改成：

```text
The proposed model is competitive with, rather than uniformly superior to, closely related hybrid reinforcement-learning portfolio frameworks.
```

### 8.2 Introduction

新增动机段落位置：当前 Introduction 第三段之后。

要点：

- 近期相近研究也使用 PPO-DQN 或 DQN + optimizer。
- 本文不直接复用原论文数据或假设。
- 本文增加平台内 clean-room reimplementation，用于在统一协议下比较结构差异。

禁止写法：

```text
We replicate their results.
We outperform the original papers.
```

建议写法：

```text
To separate algorithmic design from dataset and execution-protocol differences, we add clean-room platform-adapted reimplementations of two closely related hybrid portfolio-RL frameworks as rankable baselines.
```

### 8.3 Related Work

新增小节：

```latex
\subsection{Closest Hybrid PPO-DQN and DQN-Optimizer Frameworks}
```

内容结构：

1. 介绍 RP1：PPO-DQN hierarchical framework。
2. 介绍 RP2：DQN signal + portfolio optimization。
3. 说明它们与本文方法的差异：
   - 本文 DQN 是 binary rebalance gate。
   - RP1 复现中 DQN 是 high-level action selector。
   - RP2 复现中 DQN 是 asset signal generator。
4. 说明本文只比较平台内 clean-room reimplementation，不比较原论文原始数值。

BibTeX 接入要求：

- `paper/references.bib` 必须新增 RP1 / RP2 条目。
- citation key 建议固定为：
  - `liang2025ppodqn`：RP1, Informatica 2025。
  - `shahsafi2026hybriddqn`：RP2, Journal of Risk and Financial Management 2026。
- 元数据以 DOI、期刊官网页面或 PDF 首页为准；若页面、PDF 与 DOI 元数据不一致，提交前必须记录采用来源。
- `paper_draft.tex` 编译后不得出现 missing citation、undefined reference 或 BibTeX warning。
- 在 RP1 / RP2 没有真实 BibTeX 条目和可编译 citation 前，不得把新增 Related Work 小节标记为完成。

### 8.4 Methodology

新增小节：

```latex
\subsection{Clean-Room Reimplementations of Closest Hybrid Baselines}
```

必须包含：

- PPO-DQN hierarchy action set。
- PPO actor attribution rules。
- Hybrid DQN per-asset three-action signal。
- 五个 optimizer child。
- `portfolio_level_reward_shared=true`，不构造资产级反事实 reward。
- `algorithm_fidelity=platform_adapted`。

### 8.5 Experimental Design

在 Baselines 或 HPO 小节补充：

- 新增六个 closest-hybrid trainable entries。
- `hybrid_dqn_optimizer_reimplementation` 是 alias，不排名。
- P9 正式比较采用与主 HPO 相同 seed grid 和 validation objective。
- Smoke / pilot 只能进入 diagnostic table。

### 8.6 Results

新增小节建议放在 Main HPO Comparison 之后、Cost and Turnover 之前：

```latex
\subsection{Closest Hybrid Related-Work Reimplementations}
```

写作模板按结果分三种。

情形 A：主模型优于所有新增 baseline，且统计不显著。

```text
The proposed model ranks above the closest-hybrid reimplementations in mean cumulative return, but the pairwise evidence should be interpreted cautiously because Holm-corrected tests do not establish statistically significant superiority.
```

情形 B：主模型优于所有新增 baseline，且统计显著。

```text
Under the unified execution protocol, the proposed binary rebalancing gate significantly outperforms the closest PPO-DQN and DQN-optimizer reimplementations in paired net-return tests. This result supports the design choice of using DQN for rebalancing timing rather than for broad high-level allocation selection.
```

只有在 Holm-corrected statistics 支持时才能使用该写法。

情形 C：某个新增 baseline 优于主模型。

```text
One closest-hybrid reimplementation exceeds the proposed model in mean cumulative return. The result narrows the paper's claim: the proposed gate design remains cost-disciplined and competitive, but the broader class of hybrid PPO/DQN and DQN-optimizer designs deserves further evaluation under the unified protocol.
```

### 8.7 Discussion

新增讨论点：

- DQN 角色比较：
  - binary rebalance gate
  - high-level hierarchy selector
  - per-asset signal generator
  - direct template selector
- 如果 DQN-template 仍弱，而 Hybrid DQN optimizer 更强，说明 DQN 不适合直接输出完整组合模板，但可用于候选池或低维门控。
- 如果 PPO-DQN hierarchy 成本高于主模型，强化本文“binary gate 更适合成本控制”的解释。
- 如果 Hybrid DQN optimizer child 表现分化，说明传统 optimizer 仍是资产配置结果的重要组成部分。

### 8.8 Limitations

必须新增限制：

```text
The related-work baselines are clean-room, platform-adapted reimplementations. They do not reproduce the original papers' datasets, frequencies, transaction-cost assumptions, or reported performance numbers.
```

若 P9 只做 pilot：

```text
Smoke, fixed-config, and pilot closest-hybrid results are diagnostic. Formal P9 rows are rankable only when produced from the full equal-budget, five-seed HPO protocol.
```
