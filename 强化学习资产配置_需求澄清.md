  - 规则：正式开发目标是从零实现 `src/ [正式平台 Python 包根目录]` 下的完整实验平台；`vendor/pyfinance/legacy_rl_scripts [旧强化学习参考脚本目录]` 只作为算法思想和对比参考，不作为正式迁移目标，不要求兼容旧脚本接口；`PortfolioRebalanceEnv [目标统一 Gymnasium 资产配置环境]`、`BacktestEngine [目标统一回测引擎]`、`BaseStrategy [目标统一策略接口]` 必须按新需求重新实现。

- 数据口径与资产池闭环
  - 输入：`data/reports/*.json [本地生成清单]`、`data/**/*.parquet [本地数据产物]`、`data/processed/asset_universe.csv [标准资产池文件]`、`scripts/rebuild_etf_lof_data.py::ASSETS [数据生成资产清单]`、旧脚本 `CONFIG['fund_list'] [旧脚本内置资产列表]`。
  - 规则：默认实验以当前 `data/reports/*.json [本地生成清单]` 和实际 parquet 数据为准；原始需求中的行数、日期范围只作为历史摘要，不作为硬编码校验值；正式实现统一以 `asset_universe.csv [标准资产池文件]` 中 `status == "ok" [可用资产筛选条件]` 的资产作为默认资产池；旧脚本内置 `fund_list [旧脚本内置资产列表]` 全部废弃；`ASSETS [数据生成资产清单]` 只作为数据生成清单，不作为模型训练资产池来源。
  - 输出：`MarketDatasetBundle.data_manifest [数据口径清单]`、`run_manifest.json [运行清单]`、数据 schema 测试必须反映当前本地数据事实；`n_assets [模型资产数量]`、`availability_mask [资产可用性掩码]`、`ConstraintManager.project [组合权重约束投影]` 均以标准资产池维度运行。
  - 异常兜底：schema test 只校验字段、资产顺序、日期递增、hash、缺失规则；禁止写死旧行数；若 `asset_universe.csv [标准资产池文件]` 缺失、字段缺失、无 `status == "ok" [可用资产筛选条件]` 资产，loader 必须 fail fast 并报告缺失或不可用原因。

- 数据治理与收益字段闭环
  - 输入：`data_governance.execution_return_mode [执行收益来源模式]`、`execution_price_fields.open [开盘价矩阵]`、`execution_price_fields.close [收盘价矩阵]`、`wide_pct_chg [收盘到收盘简单收益矩阵]`、`wide_log_return [对数收益矩阵]`、`same_close_idealized_execution_enabled [同收盘价理想成交开关]`。
  - 规则：`execution_return_mode = derived_from_execution_prices [由执行价格推导收益]` 是正式执行收益口径；NAV 更新必须由 open / close 执行价格推导实际持有区间收益；`wide_pct_chg [收盘到收盘简单收益矩阵]` 只可直接支持同收盘价理想成交或明确 close-to-close 口径；`execution_price = next_open [下一交易日开盘成交]` 时，`return_from_decision_to_execution = open_{t+1} / close_t - 1 [成交前旧持仓收益]`，`holding_simple_return = close_{t+1} / open_{t+1} - 1 [成交后实际持有收益]`，不得直接用 `wide_pct_chg [收盘到收盘简单收益矩阵]` 代替；`wide_log_return [对数收益矩阵]` 仅用于模型输入、辅助标签、诊断日志或由 NAV 反推的指标校验。
  - 输出：`ExecutionMarketState.return_from_decision_to_execution [成交前旧持仓收益]`、`ExecutionMarketState.holding_simple_return [成交后实际持有收益]`、`daily_returns.csv [日度收益明细]` 使用执行价格推导口径。
  - 异常兜底：如必须从 `wide_log_return [对数收益矩阵]` 获得 simple return，使用 `simple_return_i = exp(log_return_i) - 1 [对数收益转简单收益公式]`；启用 `same_close_idealized_execution_enabled [同收盘价理想成交开关]` 时，`return_from_decision_to_execution = 0 [无成交前收益]`，`holding_simple_return = close_{t+1} / close_t - 1 [同收盘价理想化持有收益]`。

- 流动性字段闭环
  - 输入：`amount [成交额字段]`、`close [收盘价字段]`、`vol [成交量字段]`、`turnover_rate [换手率字段]`、`adv20_at_execution [执行日 20 日平均成交额]`、`feature_provenance_report.csv [特征来源审计报告]`、`run_manifest.json [运行清单]`。
  - 规则：`amount = close * vol * 100 [代理成交额公式]` 可以用于 `adv20_at_execution [执行日 20 日平均成交额]`；必须在 provenance 和 manifest 中标记为 proxy amount；`turnover_rate [换手率字段]` 全空时不 fail fast，默认删除相关特征，并在 `feature_provenance_report.csv [特征来源审计报告]` 标记 `drop_reason=all_missing [全缺失删除原因]`。
  - 输出：流动性相关模型输入、特征审计和运行清单均显式区分真实成交额与代理成交额；被删除的换手率特征进入审计报告。
  - 异常兜底：`risk_evaluation [风险评价策略]` 需要流动性特征时，默认使用 `amount / ADV20 [成交额相对 20 日均额]` 替代 `turnover_rate [换手率字段]`；只有配置显式要求 `turnover_rate_required=true [强制要求换手率配置]` 时，`turnover_rate [换手率字段]` 缺失才 fail fast。

- 环境接口与状态隔离闭环
  - 输入：`gymnasium 1.x [正式环境接口依赖]`、`PortfolioRebalanceEnv.reset [目标环境重置方法]`、`PortfolioRebalanceEnv.step [目标环境推进方法]`、`DecisionMarketState [决策阶段市场状态]`、`ExecutionMarketState [执行阶段市场状态]`。
  - 规则：正式实现只支持 `gymnasium 1.x [正式环境接口依赖]`；`reset() [目标环境重置方法]` 必须返回 `(obs, info)`；`step() [目标环境推进方法]` 必须返回 `(obs, reward, terminated, truncated, info)`；策略、agent、actor、critic、DQN gate 只能访问 `DecisionMarketState [决策阶段市场状态]`；`ExecutionMarketState [执行阶段市场状态]` 只允许 `PortfolioExecutionCore [统一组合执行核心]`、`CostModel [交易成本模型]`、`BacktestEngine [统一回测引擎]` 的执行阶段访问。
  - 输出：`observation [环境观测]`、`market_image [模型市场图像输入]`、`gate_input [DQN 门控输入]`、`strategy_state [策略状态]`、`feature_window [特征窗口]` 不得包含 `holding_simple_return [成交后实际持有收益]`、`return_from_decision_to_execution [成交前旧持仓收益]`、execution_date 后的 amount / volume / availability。
  - 异常兜底：所有 tests 必须包含 execution-only field leakage test；到达 `last_decision_index [最后可决策索引]` 时 `terminated=false [非业务终止]`、`truncated=true [split 截断结束]`；真实错误、数据耗尽或不可恢复环境状态不得伪装为 `terminated [业务终止标志]`；旧 `gym [旧环境接口依赖]` 脚本不做兼容迁移。

- 执行成交与延迟动作闭环
  - 输入：`execution_model.execution_price [执行价格模式]`、`execution_model.delayed_action_execution [延迟动作执行开关]`、`PendingAction [延迟动作对象]`、`run_manifest.json [运行清单]`。
  - 规则：正式平台默认使用 `next_open [下一交易日开盘成交]` 与 `two-stage NAV [两阶段净值公式]`；`next_open [下一交易日开盘成交]` 对应 `execution_date = date t+1 [执行日]`、`execution_price_type = open [执行价格类型]`、`next_valuation_date = date t+1 [下一估值日]`、`delayed_action_execution = false [非延迟动作]`；若 `execution_price = next_close [下一交易日收盘成交]`，必须启用 `delayed_action_execution = true [延迟动作执行]`，并使用 `PendingAction [延迟动作对象]` 记录 `decision_date [决策日]`、`execution_date [执行日]`、`execution_price_type [执行价格类型]`、`target_weights [目标权重]`、`rebalance_action [调仓动作]`、`rebalance_intensity [调仓强度]`、`decision_log_prob [决策时策略 log prob]`、`decision_value [决策时价值估计]`、`action_info [动作附加信息]`。
  - 输出：主实验 NAV、reward、turnover、transaction_cost 按实际成交区间归因；`PPO rollout [PPO 轨迹缓存]` 与 `DQN replay [DQN 回放缓存]` 必须记录 `decision_date [决策日]`、`execution_date [执行日]`、`next_valuation_date [下一估值日]`。
  - 异常兜底：若 `delayed_action_execution = false [非延迟动作]`，`execution_price [执行价格模式]` 不得配置为 `next_close [下一交易日收盘成交]`；不要求与旧脚本净值曲线保持一致；`same_close_idealized_execution_enabled [同收盘价理想成交开关]` 仅用于 legacy 对照、debug 或消融，主实验禁用；启用同收盘价理想成交时 `run_manifest.json [运行清单]` 必须写入 `idealized_execution=true [理想化成交标记]` 与 `same_close_idealized_execution_enabled=true [同收盘价理想成交标记]`。

- 两阶段 NAV 闭环
  - 输入：`decision_weights_t [决策时当前组合权重]`、`target_weights_t [目标权重]`、`executed_weights_t [执行后实际权重]`、`ExecutionMarketState.return_from_decision_to_execution [成交前旧持仓收益]`、`ExecutionMarketState.holding_simple_return [成交后实际持有收益]`、`CostModel [交易成本模型]`。
  - 规则：`pre_execution_return_t = dot(decision_weights_t, return_from_decision_to_execution) [成交前组合收益]`；`NAV_execution_t = NAV_t * (1 + pre_execution_return_t) [成交时点净值]`；`pre_execution_drifted_weights_t = drift(decision_weights_t, return_from_decision_to_execution) [执行前漂移权重]`；`turnover_t = 0.5 * sum_i |target_weights_i,t - pre_execution_drifted_weights_i,t| [换手率]`；`transaction_cost_t = CostModel(pre_execution_drifted_weights_t, target_weights_t, execution_market_state, portfolio_state) [交易成本比例]`；`NAV_after_cost_t = NAV_execution_t * (1 - transaction_cost_t) [扣成本后净值]`；`post_execution_return_t = dot(executed_weights_t, holding_simple_return) [成交后组合收益]`；`NAV_next_t = NAV_after_cost_t * (1 + post_execution_return_t) [下一估值日净值]`；`gross_return_t = (1 + pre_execution_return_t) * (1 + post_execution_return_t) - 1 [总毛收益]`；`transaction_cost_on_initial_nav_t = (NAV_execution_t * transaction_cost_t) / NAV_t [相对 step 初始净值的成本]`；`net_return_t = NAV_next_t / NAV_t - 1 [净收益]`；`portfolio_log_return_t = log(NAV_next_t / NAV_t) [组合对数收益]`。
  - 输出：`ExecutionResult [执行结果对象]`、`daily_returns.csv [日度收益明细]`、`daily_costs.csv [日度成本明细]` 必须输出上述收益、净值与成本口径字段。
  - 异常兜底：`transaction_cost_t [交易成本比例]` 默认表示相对于 `NAV_execution_t [成交时点净值]` 的成本比例；回测 NAV 必须基于真实 transaction_cost 扣除；如果 reward 已经使用 `net_log_return [扣成本净对数收益]`，不得重复扣除同一笔 transaction_cost。

- 成本资金规模闭环
  - 输入：`CostModel.estimate [交易成本估算方法]`、`ExecutionMarketState.adv20_at_execution [执行日 20 日平均成交额]`、`ExecutionMarketState.volatility_20d_at_execution [执行日 20 日波动率]`、`PortfolioState.portfolio_value [组合真实资金规模]`、`portfolio.initial_capital_currency [初始资金金额]`、`portfolio.currency [组合币种]`。
  - 规则：市场冲击成本必须使用 `portfolio_state.portfolio_value [组合真实资金规模]`，不得使用单位 NAV 替代；`portfolio_state.nav [组合单位净值]` 从 `initial_nav [初始单位净值]` 开始更新；`portfolio_state.portfolio_value [组合真实资金规模]` 从 `initial_capital_currency [初始资金金额]` 开始，并随 NAV 同比例更新；默认市场冲击公式使用 `trade_weight_i [单资产交易权重]`、`ADV20_i,t [执行日 20 日平均成交额]`、`sigma20_i,t [执行日 20 日波动率]`、`portfolio_value_t [组合真实资金规模]`；不得使用 `turnover^2 / liquidity_proxy [换手平方除以流动性代理]` 作为默认市场冲击公式。
  - 输出：`run_manifest.json [运行清单]` 必须写入 `portfolio_initial_nav [组合初始单位净值]`、`portfolio_initial_capital_currency [组合初始资金金额]`、`portfolio_currency [组合币种]`；`daily_costs.csv [日度成本明细]` 必须输出 `market_impact_cost [市场冲击成本]` 与 `total_transaction_cost [总交易成本]`。
  - 异常兜底：若 `market_impact_enabled = true [启用市场冲击成本]` 且 `portfolio_value [组合真实资金规模]` 未配置，必须 fail fast；`fixed_cost_unit [固定成本单位]` 默认为 `nav_fraction [净值比例]`，若配置为 currency，必须先转换为 NAV fraction 后进入 `transaction_cost [交易成本]`。

- 执行日可交易性与约束闭环
  - 输入：`candidate_weights [候选目标权重]`、`tradeable_mask_at_execution [执行日可交易掩码]`、`ConstraintManager.reference_weights [约束参考权重]`、`turnover_limit [换手约束]`。
  - 规则：`candidate_weights [候选目标权重]` 在 decision_date 生成后只能视为目标权重；`PortfolioExecutionCore [统一组合执行核心]` 在 execution_date 必须使用 `tradeable_mask_at_execution [执行日可交易掩码]` 二次检查并修正 `target_weights [目标权重]`；`reference_weights [约束参考权重]` 用于 `turnover_limit [换手约束]`；decision 阶段默认 `reference_weights = decision_weights_t [决策时当前组合权重]`；execution 阶段必须使用 `reference_weights = pre_execution_drifted_weights_t [执行前漂移权重]`。
  - 输出：执行日修正后的 `executed_weights [执行后实际权重]`、`realized_turnover [实际换手]`、`constraint_violations [约束违规记录]`、`execution_info [执行信息]` 进入执行结果和日度输出。
  - 异常兜底：若执行日目标资产不可交易，不得新建仓；已有持仓且不可卖出时保持冻结权重；可卖出但不可买入时允许降权、不允许增权；若配置了 `turnover_limit [换手约束]` 且 `reference_weights is None [约束参考权重为空]`，`ConstraintManager [目标约束管理器]` 必须 fail fast。

- 训练缓存日期字段闭环
  - 输入：`PPO rollout buffer [PPO 轨迹缓存]`、`DQN replay buffer [DQN 回放缓存]`、`execution_price [执行价格模式]`、`delayed_action_execution [延迟动作执行开关]`。
  - 规则：`PPO rollout buffer [PPO 轨迹缓存]` 必须保存 `decision_date [决策日]`、`execution_date [执行日]`、`next_valuation_date [下一估值日]`、`execution_price [执行价格模式]`、`delayed_action_execution [延迟动作执行开关]`；`DQN replay buffer [DQN 回放缓存]` 必须保存当前与下一状态的 `decision_date_t [当前决策日]`、`execution_date_t [当前执行日]`、`next_valuation_date_t [当前下一估值日]`、`decision_date_next [下一决策日]`、`execution_date_next [下一执行日]`、`next_valuation_date_next [下一估值日]`、`execution_price_t [当前执行价格模式]`、`delayed_action_execution_t [当前延迟执行开关]`。
  - 输出：PPO advantage / return 与 DQN gate reward 均按 action 实际执行后的收益区间对齐。
  - 异常兜底：启用 `next_close [下一交易日收盘成交]` 时，reward 与 action 归属必须按延迟动作的实际执行区间对齐；禁止使用决策日后但执行日前不可见字段构造训练输入。

- 输出 schema 冻结字段闭环
  - 输入：`run_manifest.json [运行清单]`、`daily_returns.csv [日度收益明细]`、`daily_turnover.csv [日度换手明细]`、`daily_rebalance.csv [日度再平衡明细]`、`daily_costs.csv [日度成本明细]`。
  - 规则：`run_manifest.json [运行清单]` 必须包含 `execution_model [执行模型配置]`、`data_governance [数据治理口径]`、`portfolio_initial_nav [组合初始单位净值]`、`portfolio_initial_capital_currency [组合初始资金金额]`、`portfolio_currency [组合币种]`、`execution_price [执行价格模式]`、`execution_price_type [执行价格类型]`、`delayed_action_execution [延迟动作执行开关]`、`same_close_idealized_execution_enabled [同收盘价理想成交开关]`、`idealized_execution [理想化成交标记]`、`strict_no_lookahead_execution [严格无前视执行开关]`、`t_plus_one [T+1 限制开关]`；`daily_returns.csv [日度收益明细]` 必须包含 `decision_date [决策日]`、`execution_date [执行日]`、`execution_price_type [执行价格类型]`、`next_valuation_date [下一估值日]`、`pre_execution_return [成交前收益]`、`post_execution_return [成交后收益]`、`gross_return [毛收益]`、`transaction_cost [交易成本]`、`transaction_cost_on_initial_nav [相对 step 初始净值成本]`、`net_return [净收益]`、`portfolio_log_return [组合对数收益]`、`nav [净值]`；`daily_turnover.csv [日度换手明细]` 必须包含 `decision_date [决策日]`、`execution_date [执行日]`、`execution_price_type [执行价格类型]`、`next_valuation_date [下一估值日]`、`turnover [换手]`、`rebalance_action [调仓动作]`、`rebalance_intensity [调仓强度]`、`average_holding_period [平均持有期]`；`daily_rebalance.csv [日度再平衡明细]` 必须包含 `decision_date [决策日]`、`execution_date [执行日]`、`execution_price_type [执行价格类型]`、`next_valuation_date [下一估值日]`、`rebalance_action [调仓动作]`、`rebalance_intensity [调仓强度]`、`estimated_turnover [预估换手]`、`realized_turnover [实际换手]`、`turnover [换手]`、`estimated_cost [预估成本]`、`realized_cost [实际成本]`、`q_hold [持有 Q 值]`、`q_rebalance [调仓 Q 值]`、`q_gap [调仓与持有 Q 值差]`；`daily_costs.csv [日度成本明细]` 必须包含 `decision_date [决策日]`、`execution_date [执行日]`、`execution_price_type [执行价格类型]`、`next_valuation_date [下一估值日]`、`proportional_cost [比例交易成本]`、`fixed_cost [固定成本]`、`slippage_cost [滑点成本]`、`market_impact_cost [市场冲击成本]`、`total_transaction_cost [总交易成本]`、`estimated_cost [预估成本]`、`realized_cost [实际成本]`、`turnover [换手]`。
  - 输出：日度收益、换手、再平衡、成本文件按 `date = next_valuation_date [输出日期等于下一估值日]` 对齐，用于净值曲线、日度指标和图表横轴。
  - 异常兜底：`execution_date [执行日]` 只记录交易日日期；`execution_price_type [执行价格类型]` 只记录 open / close；若输出 schema 缺少上述字段，experiment output schema tests 必须失败。

- 依赖入口闭环
