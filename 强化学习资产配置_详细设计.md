## 1. 修改摘要 (Revision Summary)

### 1.1 实现目标

本设计用于指导在 `src/ [正式平台 Python 包根目录]`、`configs/ [正式实验配置目录]`、`tests/ [正式测试目录]` 下从零实现强化学习资产配置实验平台。平台最终业务效果是：在离线 ETF / LOF 数据上，统一完成数据加载、特征审计、输入矩阵构造、训练/回测、成本扣除、约束修正、风险奖励、传统策略、深度学习策略、强化学习策略、实验归档、统计检验和自动化测试。

## 2. 业务逻辑与代码映射 (Logic Mapping)

### 2.1 配置加载与实验入口

【位置】：`src/config.py` -> `ConfigLoader` [加载 YAML、补齐默认值、禁止未知 key、生成 config_hash]。

【位置】：`src/experiments/run_experiment.py` -> `main` [单次实验命令入口]。

【执行流描述】：

前置条件：

- `config_path [配置文件路径]` 指向 `configs/` 下的 YAML。
- `security.offline_mode = true` 是默认值。
- `security.path_whitelist` 至少包含 `/Users/chenjunming/Desktop/DRL_PM`。
- `requirements.txt` 已安装，或用户显式使用 `.venv/bin/python -m pytest` / `.venv/bin/python -m src.experiments.run_experiment`。

核心步骤：

1. `main` 解析 CLI 参数：`--config`、`--seed`、`--device`、`--output`、`--run-name`。
2. `ConfigLoader.load(config_path, cli_overrides)` 读取 YAML。
3. `ConfigLoader` 使用内置 `DEFAULT_CONFIG` 对缺失 key 执行深度补齐。
4. `ConfigLoader` 对顶层 key 与 nested key 执行白名单校验。
5. 若存在未知 key，抛出 `ERR_CONFIG_UNKNOWN_KEY`，错误消息必须包含完整 key path。
6. 若 `execution_model.execution_price == "next_close"` 且 `execution_model.delayed_action_execution != true`，抛出 `ERR_CONFIG_INVALID_EXECUTION_MODEL`。
7. 若 `cost_model.market_impact_enabled == true` 且 `portfolio.initial_capital_currency is None`，抛出 `ERR_CONFIG_PORTFOLIO_VALUE_REQUIRED`。
8. 若 `device.mode == "cuda"` 但 CUDA 不可用，抛出 `ERR_DEVICE_UNAVAILABLE`。
9. 生成 `resolved_config [完整配置对象]`。
10. 生成 `config_hash = sha256(canonical_json(resolved_config))`。
11. 创建 `results/<run_name>/` 子目录。
12. 保存 `results/<run_name>/logs/config_snapshot.yaml`。
13. 调用 `registry.create_experiment(resolved_config)` 创建实验对象。
14. 调用 `experiment.run()` 执行训练、回测或聚合。
15. 结束时写入 `run_manifest.json`、metrics、figures、logs、checkpoints。

分支判断：

- 如果 `--seed` 存在，覆盖 `config.reproducibility.seed`。
- 如果 `--device` 存在，覆盖 `config.device.mode`。
- 如果 `--output` 存在，覆盖 `config.output.root`，但必须仍在 `security.path_whitelist` 下。
- 如果 `config.hpo.enabled == true`，`run_experiment.py` 调用 HPO runner；否则只执行一次 resolved config。
- 如果 `config.experiment.type == "main_model"`，只加载主模型并输出主实验产物。
- 如果 `config.experiment.type == "baseline_comparison"`，registry 加载传统策略与 `ppo_baseline`、`cnn_ppo_baseline`、`bernoulli_gated_ppo`、`dqn_only`、`eiie` 深度基准。
- 如果 `config.experiment.type == "ablation"`、`input_matrix_ablation`、`pca_ablation`、`reward_ablation`，调用消融构造器。
- 如果 `config.experiment.type == "walk_forward"`，调用 fold runner。
- 如果 `config.experiment.type == "transaction_cost_sensitivity"`，按 `cost_bps_grid=[0,5,10,20,50]` 复制实验。
- 如果 `config.experiment.type == "asset_universe_sensitivity"`，按资产池配置复制实验。
- 如果 `config.experiment.type == "market_regime"`，按市场状态切片聚合。
- 如果 `config.experiment.type == "seed_stability"`，按固定种子 `[42,123,2024,3407,9999]` 复制实验。
- 如果 `config.experiment.type == "hyperparameter_sweep"` 或 `config.hpo.enabled == true`，调用 HPO runner。
- 如果 `config.experiment.type == "auxiliary_task_sensitivity"`、`rebalance_frequency_analysis`、`preference_conditioned_analysis`、`uncertainty_analysis`、`distributional_cvar_analysis`、`partial_rebalance_analysis`、`full_reproduction`，只启用对应模块的配置分支，不改变执行、成本和输出契约。

【关键变量说明】：

- `config_path [配置文件路径]`：
  - 类型/范围：String / 必填。
  - 取值说明：必须指向当前子树内 `.yaml`。
  - 备注：路径必须通过 `Path.resolve()` 后落入 whitelist。
- `resolved_config [完整配置对象]`：
  - 类型/范围：dataclass 或 Pydantic-like validated object。
  - 取值说明：包含第 23 节所有顶层配置。
  - 备注：未知 key 必须报错，不允许静默忽略。
- `config_hash [配置哈希]`：
  - 类型/范围：String / 64 位 sha256 hex。
  - 取值说明：用于 `run_manifest.json`、`run_registry.sqlite` 和复现实验。
  - 备注：同一配置必须得到稳定 hash。
- `run_name [运行名称]`：
  - 类型/范围：String / 非空。
  - 取值说明：结果目录名。
  - 备注：只能包含字母、数字、下划线、短横线、点；否则抛出 `ERR_OUTPUT_INVALID_RUN_NAME`。

伪代码：

```python
def main(argv):
    args = parse_args(argv)
    config = ConfigLoader.load(args.config, cli_overrides=args)
    assert_path_allowed(config.output.root, config.security.path_whitelist)
    set_global_seed(config.reproducibility.seed)
    device = get_device(config.device)
    run_dir = create_run_dir(config.output.root, args.run_name)
    save_yaml(config, run_dir / "logs/config_snapshot.yaml")
    experiment = registry.create_experiment(config=config, device=device, run_dir=run_dir)
    result = experiment.run()
    write_run_manifest(result, config, run_dir)
    return result
```

### 2.2 数据加载与资产池

【位置】：`src/data/loader.py` -> `MarketDatasetBundle` [统一离线市场数据对象]。

【位置】：`src/data/loader.py` -> `load_market_dataset` [读取 `data/processed/` 与 `data/metrics_factory/`]。

【对应现状】：`scripts/rebuild_etf_lof_data.py::write_asset_universe` 生成 `asset_universe.csv`；`scripts/rebuild_etf_lof_data.py::write_wide` 生成 `wide_*.parquet`；`scripts/run_pyfinance_metrics_factory.py::combine_all_metrics` 生成 `all_metrics_features.parquet`。

【执行流描述】：

前置条件：

- `data/processed/asset_universe.csv` 存在。
- `data/processed/wide_open.parquet`、`wide_high.parquet`、`wide_low.parquet`、`wide_close.parquet`、`wide_pre_close.parquet`、`wide_pct_chg.parquet`、`wide_log_return.parquet`、`wide_amount.parquet`、`wide_vol.parquet`、`wide_turnover_rate.parquet` 存在。
- 默认 `data/metrics_factory/all_metrics_features.parquet` 存在。
- 默认 `data/reports/akshare_etf_lof_download_manifest.json` 与 `data/reports/metrics_factory_all_features_manifest.json` 存在。

核心步骤：

1. 读取 `asset_universe.csv`。
2. 校验字段：`ts_code`、`symbol`、`name`、`type`、`pool`、`status`、`rows`、`first_date`、`last_date`、`median_amount_last_252`、`raw_path`。
3. 过滤 `asset_universe.status == "ok"`，生成 `asset_ids`。
4. 如果 `asset_ids` 为空，抛出 `ERR_DATA_NO_AVAILABLE_ASSET`。
5. 按 `asset_universe.csv` 原始顺序保存 `canonical_asset_order`。
6. 对每个 required wide parquet：
   1. 读取 DataFrame。
   2. index 转为 `pd.DatetimeIndex`。
   3. 按 index 升序排序。
   4. reindex columns 到 `canonical_asset_order`。
   5. 校验列集合与顺序完全一致。
7. 读取 `etf_lof_daily_panel.parquet`，校验无重复 `(trade_date, ts_code)`。
8. 读取 `all_metrics_features.parquet`，校验无重复 `(date, ts_code)`。
9. 构造 `availability_mask = wide_close.notna() & wide_log_return.notna()`。
10. 根据 `asset_universe.first_date`、`last_date` 与每日 quote 状态构造互斥 `availability_reason`。
11. 读取 manifest，生成 `data_manifest`。
12. 返回 `MarketDatasetBundle`。

分支判断：

- 如果 `config.data.metrics_factory.enabled == false`，`metrics_features = None`，但必须记录 `data_manifest.metrics_factory_enabled = false`。
- 如果 `wide_turnover_rate.parquet` 存在但全空，loader 不报错，设置 `bundle.data_manifest["turnover_rate_all_missing"] = true`。
- 如果 `turnover_rate_required == true` 且 `wide_turnover_rate` 对可用资产全空，抛出 `ERR_DATA_TURNOVER_RATE_REQUIRED`。
- 如果 `strict_common_history_mode == true`，`date_index` 从 `common_start_date` 后开始；否则保留各资产历史，用 `availability_mask` 控制可用性。
- 如果缺失任何 required file，抛出 `ERR_DATA_MISSING_FILE`，错误消息列出所有缺失路径。
- `availability_reason` 只能取一个值，按以下优先级赋值：
  1. `trade_date < first_date`：`not_listed_yet`。
  2. `trade_date > last_date`：`delisted`。
  3. quote 存在、return 存在但成交量或成交额为 0，或 `asset_universe`/日频状态元数据明确停牌：`suspended`。
  4. `availability_mask == true`：`listed`。
  5. close/open/high/low/pre_close 任一估值必要 quote 缺失：`missing_quote`。
  6. quote 存在但 `log_return` 或 `pct_chg` 缺失：`missing_return`。
- 如果无法从现有字段识别是否停牌，不引入 `unknown`；按缺失 quote 或缺失 return 归类，并在 `data_manifest.availability_reason_inference="field_based"` 记录推断口径。

【关键变量说明】：

- `asset_universe [资产池表]`：
  - 类型/范围：`pd.DataFrame`。
  - 取值说明：每行一个资产。
  - 备注：`status == "ok"` 是默认资产池来源。
- `canonical_asset_order [标准资产顺序]`：
  - 类型/范围：`list[str]`。
  - 取值说明：来自 `asset_universe.csv` 中 ok 资产的 `ts_code` 顺序。
  - 备注：所有 wide matrix、weights、mask、模型输出都必须使用该顺序。
- `wide [宽表字典]`：
  - 类型/范围：`dict[str, pd.DataFrame]`。
  - 取值说明：key 至少包含 `open`、`high`、`low`、`close`、`pre_close`、`pct_chg`、`log_return`、`amount`、`vol`、`turnover_rate`。
  - 备注：index 为交易日，columns 为标准资产顺序。
- `availability_mask [资产可用性掩码]`：
  - 类型/范围：`pd.DataFrame`，bool。
  - 取值说明：true 表示资产在该日可参与权重归一化、风险估计和收益计算。
  - 备注：不可用资产 target weight 必须为 0。
- `availability_reason [不可用原因]`：
  - 类型/范围：`pd.DataFrame | None`，枚举字符串。

- G2 固定公式：
  1. `technical_windows = [5, 10, 20, 60, 120]`。
  2. `close_ma_ratio_w = close / rolling_mean(close, w) - 1`。
  3. `ma_5_over_20_ratio = rolling_mean(close, 5) / rolling_mean(close, 20) - 1`。
  4. `ma_20_over_60_ratio = rolling_mean(close, 20) / rolling_mean(close, 60) - 1`。
  5. `momentum_log_return_w = log(close_t / close_{t-w})`。
  6. `rolling_log_return_sum_w = rolling_sum(log_return, w)`。
  7. `rolling_volatility_w = rolling_std(pct_chg, w)`。
  8. `rolling_downside_vol_w = rolling_std(min(pct_chg, 0), w)`。
  9. `drawdown_signed_w = close / rolling_max(close, w) - 1`。
  10. `drawdown_abs_w = max(0, 1 - close / rolling_max(close, w))`。
  11. `max_drawdown_abs_w = rolling_max(drawdown_abs_w over w)`。
  12. `log1p_amount = log(1 + max(amount, 0))`。
  13. `log1p_vol = log(1 + max(vol, 0))`。
  14. `amount_ratio_20 = amount / rolling_mean(amount, 20) - 1`。
  15. `vol_ratio_20 = vol / rolling_mean(vol, 20) - 1`。
  16. `turnover_rate = turnover_rate`。
  17. `turnover_rate_ma_20 = rolling_mean(turnover_rate, 20)`。
  18. `high_low_over_close = (high - low) / close`。
  19. `close_open_over_open = (close - open) / open`。
- G4 固定公式：
  1. `risk_windows = [20, 60, 120]`。
  2. `benchmark_return_t = equal_weight_mean(available_asset_pct_chg_t)`。
  3. `rolling_beta_to_benchmark_w = cov(asset_return, benchmark_return, w) / var(benchmark_return, w)`。
  4. `rolling_corr_to_benchmark_w = corr(asset_return, benchmark_return, w)`。
  5. `rolling_cov_to_benchmark_w = cov(asset_return, benchmark_return, w)`。
  6. `asset_vol_w = rolling_std(asset_return, w)`。
  7. `asset_vol_over_benchmark_vol_w = asset_vol_w / benchmark_vol_w`。
  8. 对 rolling 60 日可用资产相关矩阵做特征分解，按资产输出 `eigen_loading_1`、`eigen_loading_2`、`eigen_loading_3`；若特征向量元素和小于 0，整体乘以 -1 固定符号。

### 2.4 预处理、特征选择与 PCA

【位置】：`src/data/feature_reduction.py` -> `FeatureReductionPipeline` [train-only imputer、scaler、feature selector、PCA]。

【执行流描述】：

前置条件：

- `feature_panel` 已生成。
- `split.train_dates`、`split.validation_dates`、`split.test_dates` 已确定。
- `availability_mask` 已独立保存。

核心步骤：

1. `FeatureReductionPipeline.fit(train_panel, feature_cols)` 只接收 train split 数据。
2. 对 train 数据估计 imputer 统计量：每个 feature 的 train median。
3. 对 train 数据估计 winsorize 上下界，默认分位数 `[0.005, 0.995]`。
4. 对 train 数据 fit `StandardScaler`。
5. 如果 `feature_selection.enabled == true`：
   1. 执行 `VarianceThreshold`，删除 train variance `< 1e-8` 的特征。
   2. 对 abs(corr) `> 0.98` 的特征对，保留 missing ratio 更低者。
   3. 若 missing ratio 相同，保留 average absolute correlation 更低者。
   4. 在 train split 内构造 `future_log_return_5d`：对每个资产使用 `sum(wide_log_return[t+1:t+5])`，标签日期窗口不得越过 validation/test 边界。
   5. 对距离 split 边界小于 `auxiliary_target_purge.horizon_days=5` 的 train 样本执行 purge，不参与 mutual information fit。
   6. 使用 purge 后 train 样本的 `future_log_return_5d` 做 mutual information filter，保留 top 512。
6. 如果 `pca.enabled == true`：
   1. 将样本展平为 train split 的 `(date, asset)` 行。
   2. fit PCA。
   3. 根据变体选取维度。
7. 对 validation / test 只调用 `transform`，禁止 fit 或更新统计量。
8. 对 PCA 输出重新拼接 `availability_mask` channel。
9. 保存 `pca_report.json`、`feature_selection_report.csv`。

分支判断：

- 如果 `input_matrix_id in {"M0", "M1", "M2", "M3", "M4", "M5"}`，不执行 PCA，除非 config 显式覆盖且实验类型是 PCA 消融。
- 如果 `input_matrix_id == "M6"`，执行 PCA，不执行 feature selection。
- 如果 `input_matrix_id == "M7"`，先 feature selection，再 PCA。
- 如果 PCA 方案 validation score 比 no_pca 差超过 1%，最终选择 no_pca。
- 如果多个 PCA 方案 validation score 距离最优不超过 1%，选择最小维度。
- 如果 `future_log_return_5d` 在 train split 内因窗口不足或 purge 后样本为空，跳过 mutual information filter 并在 `feature_selection_report.csv` 写入 `skip_reason="empty_purged_mi_label"`。

【关键变量说明】：

- `fit_scope [拟合范围]`：
  - 类型/范围：String，固定为 `train_only`。
  - 取值说明：所有统计量只在 train split fit。
  - 备注：validation / test 只 transform。
- `availability_mask_channel [可用性通道]`：
  - 类型/范围：array，shape `[1, window_size, n_assets]`。
  - 取值说明：1 表示可用，0 表示不可用。
  - 备注：不参与 imputer、scaler、selector、PCA。
- `n_pca_components [PCA 输出维度]`：
  - 类型/范围：Integer / 大于 0。
  - 取值说明：由配置或 explained variance 选出。
  - 备注：选择过程只看 validation。

### 2.6 再平衡调度器

【位置】：`src/envs/rebalance_scheduler.py` -> `RebalanceScheduler.pre_check`、`post_check`、`should_rebalance` [统一再平衡调度]。

【执行流描述】：

前置条件：

- `date [当前决策日]` 在 split date_index 中。
- `portfolio_state [组合状态]` 已更新到 decision_date close 后。
- `decision_market_state [决策阶段市场状态]` 不含 execution-only 字段。

核心步骤：

1. `pre_check` 根据无需候选权重的模式判断当前日期是否允许考虑调仓。
2. 对 `never` 永远返回 false。
3. 对 `once` 首日未建仓返回 true，之后返回 false。
4. 对 `daily` 每日返回 true。
5. 对 `weekly` / `monthly` / `quarterly` / `yearly` 判断当前日期是否是配置指定的首个或最后一个交易日。
6. 对 `every_n_days` 判断距离上次允许调仓是否达到 n 个交易日。
7. 对 `calendar_dates` 判断当前日期是否在配置列表。
8. 对 `volatility_event`、`drawdown_event`、`risk_budget_breach` 使用 `portfolio_state` 与 `decision_market_state` 判断。
9. 策略生成 `candidate_weights` 后，`post_check` 处理 `threshold_weight_drift` 与 `threshold_turnover`。
10. `should_rebalance` 返回最终 scheduler_allowed。

分支判断：

- 如果模式需要候选权重但 `candidate_weights is None`，返回 false，不报错。
- 如果策略包含 DQN gate，最终动作是 `scheduler_allowed and gate_action == 1`。
- 如果非调仓允许日，DQN gate 可不计算；若计算，只能记录诊断，不得执行交易。

【关键变量说明】：

- `scheduler_allowed_t [调度器是否允许调仓]`：
  - 类型/范围：Boolean。
  - 取值说明：true 表示当前日期允许考虑调仓。
  - 备注：不是最终交易动作。
- `gate_action_t [DQN 门控动作]`：
  - 类型/范围：Integer，0 或 1。
  - 取值说明：0 持有，1 调仓。
  - 备注：必须与 scheduler_allowed 相与。
- `final_rebalance_action_t [最终调仓动作]`：
  - 类型/范围：Integer，0 或 1。
  - 取值说明：1 表示实际进入调仓执行。
  - 备注：`1 if scheduler_allowed_t and gate_action_t == 1 else 0`。

### 2.7 组合执行核心与两阶段 NAV

【位置】：`src/envs/portfolio_execution_core.py` -> `PortfolioExecutionCore.execute_step` [统一成交、漂移、成本、NAV 更新、约束修正]。

【位置】：`src/envs/state.py` -> `PortfolioState`、`ExecutionMarketState`、`ExecutionResult`。

【执行流描述】：

前置条件：

- `decision_weights` 是 decision_date close 后可见的当前组合权重。
- `target_weights` 是策略在 decision_date 生成的候选目标权重。
- `execution_market_state` 已由 dataset 根据 `execution_model` 构造。
- `portfolio_state.nav > 0`。
- `portfolio_state.portfolio_value > 0`，若 market impact 启用则必填。

辅助函数：

- `sanitize_execution_returns(raw_returns, weights, tradeable_mask, availability_reason)`：
  1. 对 finite return 原样返回。
  2. 对 `weights_i == 0` 且 return 缺失的资产，填 0，不影响收益。
  3. 对 `weights_i > 0` 且 `tradeable_mask_i == false`，若原因是 `suspended`、`missing_quote` 或 `missing_return`，填 0 并在 `ExecutionResult.info["return_imputation"]` 记录 `{asset_id, reason, value=0}`，该资产后续卖出受冻结规则处理。
  4. 对 `weights_i > 0` 且 `tradeable_mask_i == true` 但 return 缺失，抛出 `ERR_EXECUTION_RETURN_MISSING`。
- `drift_weights(weights, simple_returns)`：
  1. 令 `gross_i = weights_i * max(1 + simple_returns_i, 0)`。
  2. 令 `denom = sum(gross_i)`。
  3. 如果 `denom <= weight_sum_eps`，抛出 `ERR_EXECUTION_INVALID_NAV`。
  4. 返回 `gross / denom`。
  5. 如果 `cash_enabled == true`，现金项使用 `simple_return=0` 一并参与归一化。

核心步骤：

1. 读取 `return_from_decision_to_execution`。
2. 调用 `sanitize_execution_returns` 生成 `r_pre`；不可用且已有持仓的缺失收益只允许按冻结资产填 0 并记录，不允许静默丢弃权重。
3. 计算 `pre_execution_return = dot(decision_weights, r_pre)`。
4. 计算 `nav_execution = nav_t * (1 + pre_execution_return)`。
5. 调用 `drift_weights(decision_weights, r_pre)` 得到 `pre_execution_drifted_weights`。
6. 若 `execution_model.t_plus_one == true`，根据 `PortfolioState.last_buy_date_per_asset` 生成 `sellable_mask`；当日买入资产在下一可卖日前不得降低权重，降低部分进入 `frozen_weight`。
7. 如果 `rebalance_action == 0`：
   1. `executed_weights = pre_execution_drifted_weights`。
   2. `turnover = 0`。
   3. `transaction_cost = 0`。
8. 如果 `rebalance_action == 1`：
   1. 使用 `tradeable_mask_at_execution` 二次修正 `target_weights`。
   2. 调用 `ConstraintManager.project(reference_weights=pre_execution_drifted_weights)`。
   3. 若 `rebalance_intensity < 1`，计算 `interpolated_weights = pre_execution_drifted_weights + rebalance_intensity * (projected_target_weights - pre_execution_drifted_weights)`。
   4. 对 `interpolated_weights` 执行 post-check：availability、long-only、simplex 必须修正；`max_weight`、`HHI`、`turnover_limit` 按 `constraints.partial_rebalance_post_check_policy` 处理。
   5. 若 policy 为 `report_only`，保留插值权重并记录 violation。
   6. 若 policy 为 `project_executed`，对插值权重再次投影并使用投影后权重成交。
   7. 若 policy 为 `force_full_rebalance`，放弃部分插值，使用 `projected_target_weights` 成交。
   8. 若 `t_plus_one == true`，对不满足 `sellable_mask` 的卖出部分恢复为 `frozen_weight`，再做 simplex 修正并记录 violation。
   9. 调用 `CostModel.estimate(pre_execution_drifted_weights, executed_weights, execution_market_state, portfolio_state)`。
   10. 得到 `turnover` 与 `transaction_cost`。
9. 计算 `nav_after_cost = nav_execution * (1 - transaction_cost)`。
10. 读取 `holding_simple_return` 并调用 `sanitize_execution_returns` 生成 `r_hold`。
11. 计算 `post_execution_return = dot(executed_weights, r_hold)`。
12. 计算 `nav_next = nav_after_cost * (1 + post_execution_return)`。
13. 如果 `nav_next <= 0`，抛出 `ERR_EXECUTION_INVALID_NAV`；`nav_next == 0` 也必须报错，因为 `portfolio_log_return` 无法定义。
14. 计算 `gross_return = (1 + pre_execution_return) * (1 + post_execution_return) - 1`。
15. 计算 `transaction_cost_on_initial_nav = nav_execution * transaction_cost / nav_t`。
16. 计算 `net_return = nav_next / nav_t - 1`。
17. 计算 `portfolio_log_return = log(nav_next / nav_t)`。
18. 更新 `portfolio_state.nav = nav_next`。
19. 更新 `portfolio_state.portfolio_value *= nav_next / nav_t`。
20. 更新 `portfolio_state.current_weights = drift_weights(executed_weights, r_hold)`，作为下一 decision date 的当前权重。
21. 更新 `PortfolioState.drifted_weights`、`previous_executed_weights`、`running_max_nav`、`current_drawdown_abs`、`rolling_returns`、`step_index`、`last_buy_date_per_asset`、`sellable_mask`、`frozen_weight`。
22. 返回 `ExecutionResult`。

分支判断：

- 如果 `execution_price == "next_open"`：
  - `execution_date = t+1`。
  - `execution_price_type = "open"`。
  - `next_valuation_date = t+1`。
  - `return_from_decision_to_execution = open[t+1] / close[t] - 1`。
  - `holding_simple_return = close[t+1] / open[t+1] - 1`。
  - `delayed_action_execution = false`。
- 如果 `execution_price == "next_close"`：
  - 必须 `delayed_action_execution = true`。
  - `execution_date = t+1`。
  - `execution_price_type = "close"`。
  - `next_valuation_date = t+2`。
  - `return_from_decision_to_execution = close[t+1] / close[t] - 1`。
  - `holding_simple_return = close[t+2] / close[t+1] - 1`。
  - 使用 `PendingAction` 对齐 reward 和 action。
- 如果 `same_close_idealized_execution_enabled == true`：
  - `return_from_decision_to_execution = 0`。
  - `holding_simple_return = close[t+1] / close[t] - 1`。
  - manifest 必须写入 `idealized_execution=true`。
- 如果 `cash_enabled == false` 且没有可用资产，抛出 `ERR_EXECUTION_NO_AVAILABLE_ASSET`。
- 如果 `partial_fill_enabled == false`，不得随机部分成交。

【关键变量说明】：

- `decision_weights [决策权重]`：
  - 类型/范围：`np.ndarray`，shape `[n_assets]`，sum 约等于 1。
  - 取值说明：策略在 decision_date close 后看到的当前组合权重。
  - 备注：不得包含现金，除非 `cash_enabled=true`。
- `pre_execution_drifted_weights [执行前漂移权重]`：
  - 类型/范围：`np.ndarray`，shape `[n_assets]`。
  - 取值说明：旧持仓经过成交前收益漂移后的权重。
  - 备注：execution 阶段 turnover_limit 的 reference_weights。
- `executed_weights [执行后权重]`：
  - 类型/范围：`np.ndarray`，shape `[n_assets]`。
  - 取值说明：实际调仓后持有权重。
  - 备注：必须满足可用性、long-only、simplex 基础约束。
- `transaction_cost [交易成本]`：
  - 类型/范围：Float，`>= 0`。
  - 取值说明：相对于 `nav_execution` 的成本比例。
  - 备注：`transaction_cost = total_transaction_cost`。
- `portfolio_value [组合真实资金规模]`：
  - 类型/范围：Float，`> 0`。
  - 取值说明：用于市场冲击成本。
  - 备注：不得用单位 NAV 替代。

伪代码：

```python
def execute_step(date, decision_weights, target_weights, rebalance_action, rebalance_intensity, execution_market_state, portfolio_state):
    nav_t = portfolio_state.nav
    r_pre = sanitize_returns(execution_market_state.return_from_decision_to_execution)
    pre_execution_return = dot(decision_weights, r_pre)
    nav_execution = nav_t * (1.0 + pre_execution_return)
    pre_drift = drift_weights(decision_weights, r_pre)

    if rebalance_action == 0:
        executed = pre_drift
        cost = zero_cost_breakdown()
    else:
        target = apply_tradeable_mask(target_weights, pre_drift, execution_market_state.tradeable_mask_at_execution)
        target = constraint_manager.project(target, execution_market_state.tradeable_mask_at_execution, portfolio_state, execution_market_state, reference_weights=pre_drift).projected_weights
        interpolated = apply_rebalance_intensity(pre_drift, target, rebalance_intensity)
        executed = post_check_partial_rebalance(interpolated, target, pre_drift, policy=config.constraints.partial_rebalance_post_check_policy)
        executed = apply_t_plus_one_freeze_if_enabled(executed, pre_drift, portfolio_state.sellable_mask)
        cost = cost_model.estimate(pre_drift, executed, execution_market_state, portfolio_state)

    nav_after_cost = nav_execution * (1.0 - cost.total_transaction_cost)
    r_hold = sanitize_returns(execution_market_state.holding_simple_return)
    post_execution_return = dot(executed, r_hold)
    nav_next = nav_after_cost * (1.0 + post_execution_return)
    return ExecutionResult(...)
```

### 2.8 回测引擎

【执行流描述】：

- `strategy` 实现 `BaseStrategy`。
- `dataset` 是 `MarketDatasetBundle` 与 `MarketImageDataset` 的组合。
- `split` 已确定。
- `PortfolioExecutionCore`、`CostModel`、`ConstraintManager` 已注入。

核心步骤：

1. 初始化 `PortfolioState`：
   1. `nav=initial_nav`。
   2. `portfolio_value=initial_capital_currency`。
   3. `current_weights=zeros(n_assets)`，表示尚未建仓的现金状态；若 `cash_enabled=true`，现金权重单独为 1。
   4. `drifted_weights=zeros(n_assets)`、`previous_executed_weights=zeros(n_assets)`。
   5. `running_max_nav=initial_nav`、`current_drawdown_abs=0`、`rolling_returns=[]`、`step_index=0`。

2. 对 split 中每个 `decision_date` 循环。
3. 构造 `DecisionMarketState`，只包含 decision_date 可见字段。
4. 调用 `RebalanceScheduler.pre_check`。
5. 如果是首个可交易决策日且 `initial_build_cost == true`，强制 `final_rebalance_action=1`，让策略目标组合从现金建仓并扣真实成本。
6. 如果是首个可交易决策日且 `initial_build_cost == false`，仍生成目标组合并建仓，但 `CostModel.estimate` 返回 0 成本且在 manifest 记录 `initial_build_cost=false`。
7. 如果 scheduler 允许或策略要求每日输出诊断，则调用 `strategy.compute_target_weights`。
8. 对需要候选权重的调度模式调用 `post_check`。
9. 合成 `final_rebalance_action`。
10. 如果 `delayed_action_execution == false`，构造当步 `ExecutionMarketState` 并立即调用 `PortfolioExecutionCore.execute_step`。
11. 如果 `delayed_action_execution == true`：
    1. 将 `decision_date`、`target_weights`、`rebalance_action`、`rebalance_intensity`、`candidate_weights`、`q_hold/q_rebalance/q_gap`、`decision_value` 写入 `PendingAction`。
    2. `PendingAction.execution_date` 由 dataset 根据 `execution_price` 解析。
    3. 将 `PendingAction` 入队。
    4. 每个循环开始先检查队首；若 `queue[0].execution_date == current_trade_date`，出队并构造对应 `ExecutionMarketState` 执行。
    5. reward、rollout item、replay item 使用 PendingAction 中保存的 decision 字段，与实际 `execution_date`、`next_valuation_date` 对齐。
12. 调用 `RewardCalculator.calculate` 生成 reward 或回测 net reward。
13. 写入内存记录：daily_returns、daily_weights、daily_turnover、daily_rebalance、daily_costs。
14. 更新 `PortfolioState`。
15. split 结束后，如果 pending queue 非空但缺少 execution 或 valuation date，按 `ERR_EXECUTION_DATE_OUT_OF_RANGE` 截断该 split，不补虚拟收益。
16. split 结束后计算 metrics。
17. 落盘 CSV、figures、manifest。

分支判断：

- 如果 `strategy.fit_required == true`，先在 train split 调用 `fit(train_data, validation_data)`。
- 如果当前是 validation / test，策略不得更新 fit 统计量。
- 如果策略优化失败，必须使用配置 fallback，并在 `action_info["fallback_reason"]` 记录。
- 如果 `ExecutionMarketState` 无法构造，抛出 `ERR_EXECUTION_DATE_OUT_OF_RANGE`。
- 如果 `initial_build_cost == true`，首日建仓的 turnover 仍按 `0.5 * sum(abs(target_weights - zero_weights))` 计算，成本从 `nav_execution` 扣除。
- 如果 `initial_build_cost == false`，只免除首日建仓成本，不改变后续真实成本扣除规则。

【关键变量说明】：

- `BacktestResult [回测结果]`：
  - 类型/范围：dataclass。
  - 取值说明：包含日度输出 DataFrame、汇总 metrics、artifact paths。
  - 备注：训练和评估共用输出 schema。
- `daily_returns [日度收益记录]`：
  - 类型/范围：`pd.DataFrame`。
  - 取值说明：一行一个 next_valuation_date。
  - 备注：`date == next_valuation_date`。

### 2.10 成本模型

【位置】：`src/envs/cost_model.py` -> `CostModel.estimate` [交易成本估算]。

【位置】：`src/envs/cost_model.py` -> `CostModel.fit_calibration` [校准成本模型分桶估计]。

【执行流描述】：

前置条件：

- `prev_weights` 是执行前漂移权重。
- `target_weights` 是执行后目标权重。
- `execution_market_state.adv20_at_execution` 与 `volatility_20d_at_execution` 已对齐资产顺序。
- `portfolio_state.portfolio_value` 已设置。
- 全局数值稳定常量固定为 `global_eps=1.0e-8`、`volatility_eps=1.0e-8`、`probability_eps=1.0e-8`、`weight_sum_eps=1.0e-12`；成本模型流动性下限固定为 `adv_eps=1000000.0`。

核心步骤：

1. 读取 `cost_model.mode`。
2. 校验 `prev_weights` 与 `target_weights` shape 一致。
3. 计算 `trade_weight = abs(target_weights - prev_weights)`。
4. 计算 `turnover = 0.5 * sum(abs(target_weights - pre_execution_drifted_weights))`，其中 `pre_execution_drifted_weights` 等于入参 `prev_weights`。
5. 计算 `proportional_cost = config.proportional_cost * turnover`。
6. 计算 `slippage_cost = config.slippage * turnover`。
7. 计算 `fixed_cost`：
   1. 如果 `fixed_cost_unit == "nav_fraction"`，直接使用配置值。
   2. 如果 `fixed_cost_unit == "currency"`，转换为 `fixed_cost_currency / portfolio_state.portfolio_value`。
8. 如果 `cost_model.mode == "empirical_default"` 且 `market_impact_enabled`：
   1. 校验 `portfolio_state.portfolio_value`。
   2. 对每个资产计算 `liquidity_ratio_i = trade_weight_i * portfolio_value / max(ADV20_i, adv_eps)`。
   3. 计算 `market_impact_cost_i = market_impact_coef * trade_weight_i * sigma20_i * sqrt(max(liquidity_ratio_i, 0))`。
   4. 汇总 `market_impact_cost`。
9. 如果 `cost_model.mode == "calibrated"`：
   1. 使用 train split 内可见成交样本构造校准表，不读取 validation/test 实际结果。
   2. 对样本按 `amount`、`turnover_rate`、`sigma20` 分桶。
   3. 每桶估计 `realized_bps_median`、`realized_bps_mean`、`sample_count`。
   4. 若当前资产落入样本数 `>= min_bucket_samples` 的桶，使用该桶 `realized_bps_median * trade_weight_i / 10000` 作为单资产校准变动成本。
   5. 若桶样本不足，按 `fallback_order = same_amount_sigma_bucket -> same_amount_bucket -> empirical_default` 回退，并记录 `fallback_reason`。
   6. `slippage_cost + market_impact_cost` 的合计由校准变动成本替代；`proportional_cost` 与 `fixed_cost` 仍按配置单独计算。
   7. 输出 `logs/cost_calibration_report.csv`。
10. `total_transaction_cost = proportional_cost + fixed_cost + slippage_cost + market_impact_cost`。
11. 返回 `CostBreakdown`。

分支判断：

- 如果 `turnover == 0`，所有成本为 0。
- 如果 `ADV20_i` 缺失或非正，使用 `adv_eps`。
- 如果 `sigma20_i` 缺失或非正，使用 `volatility_eps`。
- 如果 `market_impact_enabled == true` 且 `portfolio_value is None`，抛出 `ERR_COST_PORTFOLIO_VALUE_REQUIRED`。
- 如果 `cost_model.mode` 不在 `empirical_default | calibrated`，抛出 `ERR_CONFIG_INVALID_COST_MODE`。
- 不允许使用 `turnover^2 / liquidity_proxy` 作为默认公式。

【关键变量说明】：

- `trade_weight_i [单资产交易权重]`：
  - 类型/范围：Float，`>= 0`。
  - 取值说明：目标权重与执行前漂移权重的绝对差。
  - 备注：市场冲击按资产逐项计算。
- `ADV20_i [20 日平均成交额]`：
  - 类型/范围：Float，`>= 0`。
  - 取值说明：执行日可见的 20 日平均成交额。
  - 备注：amount 为 proxy 时必须在 provenance 和 manifest 标记。
- `sigma20_i [20 日波动率]`：
  - 类型/范围：Float，`>= 0`。
  - 取值说明：执行日 20 日波动率。
  - 备注：缺失时用 eps 兜底并记录 warning。
- `cost_calibration_report [成本校准报告]`：
  - 类型/范围：CSV。
  - 取值说明：固定字段为 `amount_bucket`、`turnover_rate_bucket`、`sigma20_bucket`、`sample_count`、`realized_bps_mean`、`realized_bps_median`、`fallback_used`、`fallback_reason`、`status`。
  - 备注：始终输出；`empirical_default` 写入 `status="not_applicable"`，`calibrated` 样本不足时写入回退记录。

### 2.11 约束管理器

【位置】：`src/envs/constraint_manager.py` -> `ConstraintManager.project` [组合权重投影]。

【执行流描述】：

前置条件：

- `raw_weights` shape 为 `[n_assets]`。
- `available_mask` shape 为 `[n_assets]`。
- 如果配置 `turnover_limit`，必须提供 `reference_weights`。

核心步骤：

1. 校验输入 shape、finite。
2. 运行 feasibility check。
3. 将 unavailable asset 权重强制设为 0。
4. 对剩余权重执行 long-only：负数截断为 0。
5. 若没有 available assets：
   1. 如果 `cash_enabled == true`，转入现金。
   2. 否则抛出 `ERR_CONSTRAINT_NO_AVAILABLE_ASSET`。
6. 执行 simplex 归一化。
7. 应用 max_weight。
8. 应用 min_weight。
9. 如果配置 turnover_limit，使用 `reference_weights` 投影到 turnover ball。
10. 如果配置 HHI limit，投影或放宽到理论可行下限。
11. 如果配置 asset-class exposure limit：
   1. 优先读取 `constraints.asset_class_mapping[ts_code]`。
   2. 若配置未提供，则读取 `asset_universe.pool`；若 `pool` 为空再读取 `asset_universe.type`。
   3. 对每个 class 汇总 `sum(weights_i)`，与 `min_exposure/max_exposure` 比较。
   4. 元数据不存在且 `constraints.asset_class_required == false` 时跳过该约束并记录 `skip_reason="missing_asset_class_metadata"`。
   5. 元数据不存在且 `constraints.asset_class_required == true` 时抛出 `ERR_CONSTRAINT_ASSET_CLASS_METADATA_REQUIRED`。
12. 软风险约束只记录 penalty 或 violation，不破坏 simplex。
13. 如果 `constraint_method == "ppo_lagrangian"`：
   1. availability、long-only、simplex、T+1 不作为软约束，仍硬修正。
   2. 对 max_weight、HHI、turnover、asset-class exposure 计算 violation scalar。
   3. 在 PPO loss 中加入 `sum(lambda_k * violation_k)`。
   4. 每个 train update 后执行 `lambda_k = max(0, lambda_k + lagrangian_lr * mean_violation_k)`。
   5. validation/test 不更新 lambda，只记录 violation。
14. 返回 `ConstraintResult`。

分支判断：

- 如果 `max_weight * n_available < 1`，放宽 `max_weight = 1 / n_available`，记录 violation。
- 如果 `min_weight * n_available > 1`，放宽 `min_weight = 0`，记录 violation。
- 如果 `HHI_limit < 1 / n_available`，放宽到 `1 / n_available`，记录 violation。
- 如果 `turnover_limit` 配置且 `reference_weights is None`，抛出 `ERR_CONSTRAINT_REFERENCE_WEIGHTS_REQUIRED`。
- unavailable asset 权重优先为 0，优先级高于 min_weight。
- `constraint_method == "hard_projection"` 时使用投影结果作为成交权重。
- `constraint_method == "soft_penalty"` 时基础硬约束仍投影，软约束只进入 reward/diagnostic penalty。
- `constraint_method == "ppo_lagrangian"` 只影响 PPO 训练损失，不改变 BacktestEngine 的最终执行契约。

【关键变量说明】：

- `reference_weights [约束参考权重]`：
  - 类型/范围：`np.ndarray | None`。
  - 取值说明：turnover_limit 的比较基准。
  - 备注：decision 阶段默认 `decision_weights_t`；execution 阶段必须是 `pre_execution_drifted_weights_t`。
- `constraint_violations [约束违规记录]`：
  - 类型/范围：`list[dict]`。
  - 取值说明：记录被修正、放宽或无法满足的约束。
  - 备注：必须进入 `ExecutionResult.info` 与日度输出。

### 2.12 奖励计算

【位置】：`src/envs/reward_calculator.py` -> `RewardCalculator.calculate` [奖励函数]。

【执行流描述】：

前置条件：

- `ExecutionResult` 已包含 `net_return`、`portfolio_log_return`、`turnover`、`transaction_cost`。
- `PortfolioState` 已有 rolling returns、drawdown、risk state。

核心步骤：

1. 根据 reward variant 选择计算分支。
2. 默认使用 `net_log_return_t = execution_result.portfolio_log_return`。
3. 计算 `downside_penalty_t = max(0, target_return_daily - net_return_t)^2`。
4. 计算 `drawdown_penalty_t = max(0, drawdown_abs_t - drawdown_threshold)`。
5. 计算 `rolling_volatility_annual_t = rolling_std(net_return, volatility_window) * sqrt(252)` 与 `volatility_penalty_t = max(0, rolling_volatility_annual_t - volatility_threshold_annual)`。
6. 计算 `CVaR_loss_t = max(0, -mean(lower_tail_returns))` 与 `cvar_penalty_t = max(0, CVaR_loss_t - CVaR_loss_threshold)`。
7. 计算 `concentration_penalty_t = max(0, HHI_t - HHI_threshold)`。
8. 默认汇总 `reward = net_log_return_t - lambda_turnover * turnover_t - lambda_downside * downside_penalty_t - lambda_drawdown * drawdown_penalty_t - lambda_volatility * volatility_penalty_t - lambda_cvar * cvar_penalty_t - lambda_concentration * concentration_penalty_t`。
9. 对 `A7_differential_sharpe` 使用 episode 内 A/B 递推状态。
10. 返回 reward 与 reward_info。

分支判断：

- 如果 reward variant 是 `A0_raw_simple_return`，使用 gross or raw simple return，但回测 NAV 仍必须扣真实成本。
- 如果 reward variant 是 `A1_log_return`，使用 `gross_log_return_t`，不额外扣 cost penalty。
- 如果 reward variant 是 `A2_net_log_return_after_cost`，不得在 reward 中二次扣 transaction_cost。
- 如果 reward variant 是 `A3_net_log_return_plus_turnover`，使用 `net_log_return_t - lambda_turnover * turnover_t`。
- 如果 reward variant 是 `A4_net_log_return_plus_turnover_downside`，在 A3 基础上扣 `lambda_downside * downside_penalty_t`。
- 如果 reward variant 是 `A5_net_log_return_plus_turnover_drawdown`，在 A3 基础上扣 `lambda_drawdown * drawdown_penalty_t`。
- 如果 reward variant 是 `A6_net_log_return_plus_turnover_downside_drawdown`，在 A3 基础上同时扣 downside 与 drawdown penalty。
- 如果 reward variant 是 `A7_differential_sharpe`，使用差分夏普递推奖励。
- 如果 reward variant 是 `A8_cvar_sensitive`，在默认 reward 中启用 CVaR penalty 且 `cvar_confidence=0.95`。
- 如果 reward variant 是 `A9_benchmark_relative`，使用 `net_log_return_t - benchmark_log_return_t` 作为基础项，其他 penalty 按配置叠加。
- 如果 reward variant 是 `A10_ppo_lagrangian`，约束 violation 通过 PPO-Lagrangian multiplier 进入训练损失，环境 NAV 和成本不变。
- 如果 reward variant 是 `A11_regime_aware`，按 market regime 选择参数组；缺失 regime 时使用默认参数组。
- 如果 reward variant 是 `A12_multi_objective_preference_conditioned`，构造 reward vector 并与 `omega` 点乘。
- 如果 `preference_conditioned` 启用，构造 reward vector 并用 omega 点乘。
- 如果 warm-up 不足 20 step 且使用 differential Sharpe，reward 使用 net_log_return。
- 默认 reward 参数固定为 `lambda_turnover=0.001`、`lambda_downside=0.10`、`lambda_drawdown=0.20`、`lambda_volatility=0.05`、`lambda_cvar=0.10`、`lambda_concentration=0.02`、`target_return_daily=0.0`、`drawdown_threshold=0.10`、`volatility_window=20`、`volatility_threshold_annual=0.25`、`cvar_window=60`、`cvar_confidence=0.95`、`hhi_threshold=0.20`。
- `A7_differential_sharpe` 状态固定为 `A_0=0`、`B_0=0`、`eta=1/252`、`differential_sharpe_eps=1.0e-8`；每个 episode reset `A_t/B_t`，前 20 step reward 使用 `net_log_return_t`。

【关键变量说明】：

- `net_log_return_t [扣成本净对数收益]`：
  - 类型/范围：Float。
  - 取值说明：`log(NAV_next / NAV_t)`。
  - 备注：默认 reward 基础项。
- `CVaR_loss_t [尾部损失]`：
  - 类型/范围：Float，`>= 0`。
  - 取值说明：低尾收益均值取负后的损失口径。
  - 备注：`cvar_confidence=0.95` 对应 `cvar_alpha=0.05`。
- `reward_variant [奖励变体]`：
  - 类型/范围：String，枚举 `A0_raw_simple_return` 至 `A12_multi_objective_preference_conditioned`。
  - 取值说明：控制基础收益项和 penalty 组合。
  - 备注：无论 reward variant 如何变化，回测 NAV 必须使用真实成本。

### 2.13 传统策略

【位置】：`src/baselines/base_strategy.py` -> `BaseStrategy` [统一策略接口]。

【位置】：`src/baselines/*.py` -> 各传统策略实现。

【执行流描述】：

前置条件：

- 每个传统策略只能访问 `DecisionMarketState`。
- 每个传统策略必须返回 `PortfolioAction`。
- 每个传统策略的目标权重必须经过 `ConstraintManager`。

核心步骤：

1. `BacktestEngine` 调用 `strategy.compute_target_weights(date, decision_market_state, portfolio_state, available_mask)`。
2. 策略根据 lookback window 取 decision_date 之前的数据。
3. 策略计算 raw weights。
4. raw weights 对 unavailable asset 置 0。
5. 调用 `ConstraintManager.project(reference_weights=portfolio_state.current_weights)`。
6. 返回 `PortfolioAction(projected_weights, rebalance_action, rebalance_intensity, action_info)`。

分支判断：

- `fixed_ratio`：按配置资产权重或资产类别权重生成目标；类别无可用资产时按其他可用类别重分配。
- `equal_weight`：所有当前可用资产权重 `1 / n_available`。
- `buy_and_hold`：首日建仓后默认返回保持，资产不可用或 forced_rebalance 时再调仓。
- `traditional_markowitz_mean_variance`：最大化 `mu^T w - lambda_risk * w^T Sigma w`。
- `markowitz_min_variance`：最小化 `w^T Sigma w`。
- `markowitz_max_sharpe`：最大化 `(mu^T w - risk_free_rate) / sqrt(w^T Sigma w)`。
- `risk_parity`：最小化风险贡献偏离。
- `inverse_volatility`：`raw_w_i = 1 / (vol_i + eps)`。
- `minimum_drawdown`：用 SLSQP 或 random search 求低回撤组合。
- `risk_evaluation`：按 return、volatility、MDD、downside、CVaR、liquidity 打分。
- `HRP`：相关性距离、层次聚类、quasi-diagonalization、递归二分。
- `momentum`：按 momentum 或 risk-adjusted momentum 排序选择 top_k。
- `risk_evaluation` 的 `liquidity_i` 来源：
  1. 若 `DecisionMarketState.turnover_rate` 存在且非全空，使用 train-only rank/z-score 后的 `turnover_rate_i`。
  2. 若 `turnover_rate` 缺失或全空，使用 `amount_i / max(ADV20_i, adv_eps)`。
  3. 若 `amount_i` 或 `ADV20_i` 缺失，使用该资产 train split 内 liquidity 中位数。
  4. 若 train split 内仍无法估计，记录 `fallback_reason="liquidity_unavailable"` 并将 liquidity score 置为横截面中性值 0。

【关键变量说明】：

- `lookback_window [历史估计窗口]`：
  - 类型/范围：Integer，`> 0`。
  - 取值说明：策略只使用 decision_date 之前的历史。
  - 备注：参数只能用 validation 选择。
- `fallback [失败兜底策略]`：
  - 类型/范围：String。
  - 取值说明：如 `equal_weight`、`inverse_volatility`。
  - 备注：优化失败时必须记录 `fallback_reason`。

### 2.14 深度模型、强化学习模型与门控

【位置】：`src/models/encoders.py` -> `Encoder` family [状态编码]。

【位置】：`src/models/ppo_actor.py` -> `PPOActor` [连续权重策略]。

【位置】：`src/models/dqn_gate.py` -> `DQNGate` [调仓门控]。

【位置】：`src/models/*_gated_ppo.py` -> 五类强化学习算法模块。

【执行流描述】：

前置条件：

- `state_t` 只包含 `market_image`、`current_weights`、`availability_mask`、可选 `risk_state`。
- `availability_mask` 已在 distribution 构造阶段使用。
- `DQN gate` 只能使用 decision_date 可见字段。
- 默认模型参数固定为 `model.default_encoder="cnn"`、`model.latent_dim=256`、`model.dropout=0.10`、`model.activation="GELU"`、`model.normalization="LayerNorm"`。

核心步骤：

1. `Encoder` 接收 `market_image`，输出 `z_t`。
2. `PPOActor` 根据 `z_t` 与可选状态输出 Dirichlet alpha。
3. 在 available assets 子空间构造 Dirichlet。
4. 训练时 sample candidate weights；评估时用 alpha mean。
5. 将 available weights scatter 回 `[n_assets]`，unavailable asset 权重为 0。
6. 估算 `estimated_turnover_t = 0.5 * sum(abs(candidate_w - decision_weights))`。
7. 用 `DecisionMarketState` 可见字段估算 `estimated_cost_t`。
8. `DQNGate` 输入 `z_t`、`decision_weights_t`、`candidate_w_t`、`estimated_turnover_t`、`estimated_cost_t`、risk optional。
9. 输出 hold / rebalance 或 partial rho。
10. 环境执行后记录 realized turnover / realized cost。
11. PPO、DQN、auxiliary heads 分别按训练规则更新。

分支判断：

- PPO baseline：无 DQN gate，candidate weights 直接进入执行，仍受 scheduler 和约束控制。
- CNN-PPO：Encoder 使用 CNN。
- Bernoulli-Gated CNN-PPO：gate 使用 on-policy Bernoulli log_prob。
- DQN-Only：DQN 输出离散模板编号，模板函数生成连续权重。
- EIIE：共享 evaluator 对每资产输出 score，softmax 后生成权重。
- Full DQN-Gated Multi-Task CNN-PPO：PPO 候选权重 + DQN 是否调仓 + auxiliary heads。
- Preference-Conditioned：omega 进入 actor、critic、gate，reward vector 点乘 omega。
- Uncertainty-Aware：多 head 或 dropout 生成 candidate weight variance 与 q_uncertainty。
- Distributional-CVaR：distributional critic 输出 quantiles，gate 输入 CVaR risk features。
- Partial-Rebalance：DQN 离散 rho 或 Beta actor 连续 rho，执行插值。
- `estimated_cost_t` 估算公式：
  1. `estimated_trade_weight_i = abs(candidate_w_i - decision_weights_i)`。
  2. `estimated_turnover_t = 0.5 * sum(estimated_trade_weight_i)`。
  3. `estimated_proportional_cost_t = proportional_cost * estimated_turnover_t`。
  4. `estimated_slippage_cost_t = slippage * estimated_turnover_t`。
  5. `estimated_fixed_cost_t` 按 `fixed_cost_unit` 使用 decision_date 可见的 `portfolio_state.portfolio_value` 换算。
  6. `estimated_liquidity_ratio_i = estimated_trade_weight_i * portfolio_state.portfolio_value / max(ADV20_i_at_decision, adv_eps)`。
  7. `estimated_market_impact_cost_i = market_impact_coef * estimated_trade_weight_i * sigma20_i_at_decision * sqrt(max(estimated_liquidity_ratio_i, 0))`。
  8. `estimated_cost_t = estimated_proportional_cost_t + estimated_fixed_cost_t + estimated_slippage_cost_t + sum(estimated_market_impact_cost_i)`。
  9. 若 `cost_model.mode="calibrated"`，只使用 train-fitted calibration table 和 decision_date 可见桶，不使用 execution_date 实际成交信息。
  10. `estimated_cost_t` 只进入 gate 输入和诊断；回测 NAV 只使用 execution 阶段的 `realized_cost`。
- 默认架构固定参数：
  1. CNN encoder：`cnn_channels=[32,64,128]`、`kernel_size_time=3`、`kernel_size_asset=3`、`stride=1`、`pooling="adaptive_average"`、`output_latent_dim=256`。
  2. cross-asset attention：`enabled=false`、`n_heads=4`、`n_layers=1`、`dropout=0.10`。
  3. temporal transformer：`enabled=false`、`n_layers=2`、`n_heads=4`、`model_dim=256`、`feedforward_dim=512`、`dropout=0.10`。
  4. TCN：`enabled=false`、`channels=[64,128,128]`、`kernel_size=3`、`dilation_base=2`。
  5. PPO actor：`hidden_dims=[256,128]`、`distribution="Dirichlet"`、`min_alpha=0.001`。
  6. PPO critic：`hidden_dims=[256,128]`。
  7. DQN gate：`hidden_dims=[256,128]`、`output_dim=2`，其中 `0=hold`、`1=rebalance`。
  8. auxiliary heads：`hidden_dims=[128]`。
- PPO baseline MLP encoder 固定结构：
  1. 输入为 `flatten(market_image) + current_weights + availability_mask`。
  2. `Linear(input_dim,512) -> LayerNorm(512) -> GELU -> Dropout(0.10)`。
  3. `Linear(512,256) -> LayerNorm(256) -> GELU -> Dropout(0.10)`。
  4. `Linear(256,256) -> LayerNorm(256) -> GELU`。
  5. 输出 `latent_dim=256`。
- auxiliary target 固定清单：
  1. `future_log_return_5d = log(close_{t+5} / close_t)`。
  2. `future_volatility_20d = std(simple_return_{t+1:t+20})`。
  3. `future_trend_10d = 1 if log(close_{t+10} / close_t) > 0 else 0`。
  4. `future_cross_sectional_rank` 为同一 date 横截面未来收益 rank。
  5. `future_downside_volatility` 为未来窗口负收益 downside volatility。
  6. `future_max_drawdown` 为未来窗口最大绝对回撤。
  7. `future_CVaR` 为未来窗口 lower-tail CVaR loss。
  8. `future_correlation_or_covariance` 为未来窗口低维 covariance/correlation embedding。
  9. `masked_feature_reconstruction` 为 masked market image 重建目标。
- auxiliary loss 固定口径：
  1. future return 使用 Huber 或 MSE。
  2. future volatility 使用 `log(volatility + volatility_eps)` 上的 Huber 或 MSE。
  3. future trend 使用 `BCEWithLogitsLoss` 或 3-class cross entropy。
  4. rank target 使用 pairwise rank loss 或 cross-sectional MSE。
  5. covariance target 使用低维 covariance embedding 的 MSE。
  6. masked features 使用 reconstruction loss。
  7. `L_representation_regularization = mean(z_t ** 2)`，不 detach，允许回传到 encoder。
  8. `L_aux = 0.10 * L_future_return + 0.05 * L_future_trend + 0.05 * L_future_volatility + 0.10 * L_future_rank + 0.05 * L_future_downside_volatility + 0.05 * L_future_max_drawdown + 0.05 * L_future_CVaR + 0.03 * L_future_covariance + 0.05 * L_masked_feature + 0.005 * L_representation_regularization`。
  9. 若 `z_t` shape 为 `[batch, latent_dim]`，对全部元素取 mean；若 shape 为 `[batch, n_assets, latent_dim]`，只对 `availability_mask == true` 的资产取 mean。
  10. future target 窗口跨越 split 边界的样本必须 purge；auxiliary target 不得进入 `feature_cols`、observation 或模型输入特征。

【关键变量说明】：

- `alpha [Dirichlet 浓度参数]`：
  - 类型/范围：positive float tensor。
  - 取值说明：每个可用资产一个 alpha。
  - 备注：`softplus(raw) + min_alpha`；NaN/Inf fail fast。
- `candidate_weights [候选权重]`：
  - 类型/范围：float array，shape `[n_assets]`。
  - 取值说明：PPO actor 或模板策略生成的目标权重。
  - 备注：decision_date 生成后只能视为目标，execution_date 还要二次可交易检查。
- `rebalance_intensity [调仓强度]`：
  - 类型/范围：Float，`0 <= rho <= 1`。
  - 取值说明：0 不调仓，1 完全调仓，中间值部分调仓。
  - 备注：部分调仓后必须 feasibility check。
- `omega [偏好向量]`：
  - 类型/范围：float array，shape `[5]`。
  - 取值说明：return、turnover、drawdown、cvar、concentration 权重。
  - 备注：非负且 sum 为 1。

### 2.15 训练缓存与训练循环

【位置】：`src/buffers/rollout_buffer.py` -> `RolloutBuffer` [PPO 轨迹缓存]。

【位置】：`src/buffers/replay_buffer.py` -> `ReplayBuffer` [DQN 回放缓存]。

【位置】：`src/agents/hybrid_agent.py` -> `HybridAgent.train` [混合训练循环]。

【执行流描述】：

前置条件：

- `PortfolioRebalanceEnv` 可运行。
- 模型、优化器、scheduler、device 已初始化。
- PPO 默认超参数固定为 `rollout_steps=256`、`minibatch_size=64`、`update_epochs=10`、`gamma=0.99`、`gae_lambda=0.95`、`clip_range=0.20`、`value_clip_range=0.20`、`entropy_coef=0.01`、`value_coef=0.50`、`max_grad_norm=0.50`、`advantage_normalization=true`。
- DQN 默认超参数固定为 `replay_size=100000`、`batch_size=128`、`warmup_steps=1000`、`gamma=0.99`、`target_update_interval=500`、`soft_update_tau=null`、`epsilon_start=1.00`、`epsilon_end=0.05`、`epsilon_decay_steps=20000`、`use_double_dqn=true`、`use_dueling=true`、`use_noisynet=false`、`use_n_step=true`、`n_steps=3`、`use_prioritized_replay=true`、`per_alpha=0.60`、`per_beta_start=0.40`、`per_beta_end=1.00`、`per_priority_eps=1.0e-6`。
- Optimizer 默认固定为 `AdamW`，`betas=[0.9,0.999]`、`eps=1.0e-8`、`weight_decay=1.0e-4`、`ppo_lr=3.0e-4`、`dqn_lr=1.0e-4`、`auxiliary_lr=3.0e-4`。
- Scheduler 默认 `enabled=false`；可选 `optional_type="cosine_with_warmup"`、`warmup_ratio=0.05`、`min_lr_ratio=0.10`。
- Checkpoint 选择固定为 `primary=validation_sharpe`、`tie_breaker_1=lower_validation_max_drawdown`、`tie_breaker_2=lower_validation_turnover`、`tie_breaker_3=earliest_global_step`。

核心步骤：

1. collect on-policy rollout。
2. 每个 step 保存 `decision_date`、`execution_date`、`next_valuation_date`、`execution_price`、`delayed_action_execution`。
3. `RolloutItem` 固定保存 `state`、`candidate_weights`、`log_prob`、`value`、`decision_value`、`executed_weights`、`gate_action`、`rebalance_action`、`rebalance_intensity`、`reward`、`terminated`、`truncated`、`advantage`、`return`、`auxiliary_labels`、`preference_vector`、`uncertainty_features`、`distributional_features`。
4. `ReplayItem` 固定保存 `state_t`、`state_tp1`、`decision_date_t`、`execution_date_t`、`next_valuation_date_t`、`execution_price_t`、`candidate_weights_t`、`executed_weights_t`、`gate_action_t`、`rebalance_action_t`、`rebalance_intensity_t`、`estimated_turnover_t`、`realized_turnover_t`、`estimated_cost_t`、`realized_cost_t`、`reward_t`、`terminated_t`、`truncated_t`、`q_hold_t`、`q_rebalance_t`、`q_gap_t`。
5. 同步将 DQN transition 写入 replay buffer。
6. rollout 满 `rollout_steps` 后计算 GAE。
7. 对 PPO actor / critic 执行 clipped objective。
8. 对 DQN gate 执行 Double DQN update。
9. 如果 PER 启用，更新 priority。
10. 更新 auxiliary heads。
11. 根据共享 encoder gradient rule 更新 encoder。
12. 在 validation split deterministic evaluation。
13. 根据 checkpoint_selection 保存 best 和 last。
14. `save_checkpoint(model_dict, path)` 写入完整恢复训练状态。
15. `load_checkpoint(path, device)` 加载到指定 device，并恢复训练或 deterministic evaluation 所需状态。

分支判断：

- gate=hold 样本默认不进入 PPO actor policy loss，但进入 critic、encoder、auxiliary。
- partial rho 为 0 时 candidate weights 不进入 actor loss。
- partial 0 < rho < 1 时 actor_loss_weight = rho。
- delayed action 启用时，reward 与 action 按 `PendingAction` 实际执行区间对齐。
- episode terminal、truncation、split boundary 均截断 N-step。
- PPO actor loss 固定为 clipped objective：`min(ratio_t * advantage_t, clip(ratio_t, 1 - clip_range, 1 + clip_range) * advantage_t)`。
- PPO value loss 在 `value_clip_range` 启用时使用 clipped value loss；advantage normalization 为默认 true。
- DQN 使用 Double DQN target：`target = reward + gamma^n * Q_target(next_state, argmax_a Q_online(next_state,a))`，terminal/truncated/split boundary 时不 bootstrap。
- PER 规则固定为 `priority_i = (abs(td_error_i) + per_priority_eps) ** per_alpha`、`sampling_probability_i = priority_i / sum(priority)`、`IS_weight_i = (N * sampling_probability_i) ** (-beta)`、`IS_weight_i = IS_weight_i / max(IS_weight in batch)`；`beta` 从 `per_beta_start` 线性增长到 `per_beta_end`。
- N-step 规则固定为 episode terminal、truncation、split boundary 均截断；rollout batch end 但 episode 未结束时 pending n-step buffer 可延续。
- 默认 encoder 由 PPO loss + auxiliary loss 更新；DQN 默认 detach encoder latent。若允许 DQN 更新 shared encoder，必须作为消融配置。
- checkpoint 必须包含 `encoder_state`、`ppo_actor_state`、`ppo_critic_state`、`dqn_gate_state`、`dqn_target_network_state`、`auxiliary_head_states`、`optimizer_states`、`scheduler_states`、`amp_grad_scaler_state`、`replay_buffer_state`、`epoch`、`global_step`、`best_validation_metric`、`rng_states`、`resolved_config`。
- `rng_states` 必须包含 Python `random`、NumPy、Torch CPU、CUDA 可用时的 CUDA RNG state、环境随机状态。
- CUDA AMP 启用时 checkpoint 必须包含 GradScaler state；CPU/MPS 时该字段为 null。

【关键变量说明】：

- `RolloutItem [PPO 单步样本]`：
  - 类型/范围：dataclass。
  - 取值说明：需求第 18 节列出的所有字段。
  - 备注：日期字段必填。
- `ReplayItem [DQN 单步样本]`：
  - 类型/范围：dataclass。
  - 取值说明：包含当前和下一状态的 decision/execution/valuation 日期。
  - 备注：不得丢失 execution_price 与 delayed_action_execution。

  - 类型/范围：dict。
  - 取值说明：完整恢复训练和评估所需的模型、优化器、scheduler、AMP、replay buffer、step、metric、RNG、resolved config。
  - 备注：best 与 last checkpoint 使用同一 schema。

### 2.17 实验矩阵、HPO、消融与 Walk-forward

【位置】：`src/experiments/registry.py` -> `ExperimentRegistry.create_experiment` [实验类型路由]。

【位置】：`src/experiments/aggregate_results.py` -> `aggregate_walk_forward` [fold 与全样本外聚合]。

【执行流描述】：

前置条件：

- `split` 已由 `src/data/splits.py` 生成。
- `config.experiment.type` 属于第 20 节冻结枚举。
- 每个实验分支共享同一个 `PortfolioExecutionCore`、`CostModel`、`ConstraintManager` 和输出 schema。

HPO 核心步骤：

1. 读取 `config.hpo.enabled`、`study_name`、`storage`、`sampler="tpe"`、`pruner="median"`、`n_trials`、`timeout`、`metric`、`direction`、`seed`。
2. 使用 Optuna TPE sampler 创建 study；sampler seed 等于实验 seed。
3. 使用 median pruner；warmup trial 数、warmup step 数从 `config.hpo.pruner` 读取。
4. 每个 trial 只训练 train split，并只用 validation split 计算 objective。
5. 每个 trial 固定相同 train/validation/test 切分、相同 seed grid、相同成本扣除规则。
6. trial 内禁止读取 test split 指标；test split 只在 best trial 锁定后执行一次。
7. 对每个 trial 写入 `logs/hpo_trials.csv`，字段至少包含 `study_name`、`trial_number`、`seed`、`params_json`、`state`、`objective_value`、`validation_metric`、`train_start`、`train_end`、`duration_sec`、`pruned_step`、`fail_reason`。
8. 若 trial 训练异常，记录 `state="fail"` 与 `fail_reason`，不吞掉不可恢复的数据契约错误。
9. best trial 的 config 写入 `logs/config_snapshot.yaml` 和 `run_manifest.json.best_trial_number`。

消融核心步骤：

1. 读取 `config.ablation.base_config`。
2. 每个 ablation run 只切换一个开关或一个参数族，生成 `ablation_id` 与 `changed_key_path`。
3. `input_matrix_ablation` 只修改 `feature_matrix.input_matrix_id`。
4. `pca_ablation` 只修改 `pca.enabled` 或 `pca.n_components`。
5. `reward_ablation` 只修改 reward variant 或某一个 penalty 系数。
6. `auxiliary_task_sensitivity` 只修改 auxiliary head 开关或 loss weight。
7. 所有消融无论 reward 是否去掉成本惩罚，回测 NAV 均必须扣真实 `transaction_cost`。
8. 风险惩罚消融只移除 reward penalty，不移除 risk 指标、risk_state 诊断或输出字段。
9. 输出 `metrics/ablation_results.csv`、`metrics/input_matrix_ablation_results.csv`、`metrics/PCA_ablation_results.csv`、`metrics/reward_ablation_results.csv` 中对应文件。

Walk-forward 核心步骤：

1. `create_split(config.split.method="walk_forward")` 生成有序 fold 列表。
2. 对每个 `fold_id`：
   1. fit imputer/scaler/selector/PCA/模型/HPO 只使用该 fold train。
   2. model selection 只使用该 fold validation。
   3. test 只执行该 fold 的 out-of-sample 日期段。
   4. 输出 fold 级 `daily_returns`、`daily_weights`、`daily_turnover`、`daily_rebalance`、`daily_costs`，所有行带 `fold_id`。
   5. 计算 fold 级 metrics，写入 `metrics/walk_forward_results.csv`。
3. fold 结束后，将所有 fold 的 out-of-sample daily net returns 按 `date` 升序拼接。
4. 如果不同 fold 日期重叠，按 fold 生成顺序保留第一条并记录 `duplicate_oos_date_count`。
5. 基于拼接后的全样本外序列计算整体 CAGR、annual volatility、Sharpe、Sortino、Calmar、max drawdown、CVaR、turnover、cost、hit ratio。
6. 将 fold 级指标和拼接全样本外指标同时写入 `metrics/walk_forward_results.csv`；全样本外行使用 `fold_id="all_oos"`。

分支判断：

- 如果 HPO 与 walk-forward 同时启用，HPO 必须在每个 fold 内独立运行，不能用后续 fold 或全样本 test 信息。
- 如果 HPO 与 seed stability 同时启用，trial budget 对每个候选模型相同，seed grid 相同，保证公平性。
- 如果 ablation 配置同时修改多个非同族 key，抛出 `ERR_EXPERIMENT_ABLATION_NOT_SINGLE_SWITCH`。
- 如果 `full_reproduction` 启用，按主模型、基准、消融、敏感性、walk-forward、统计检验顺序执行，但仍复用相同模块，不新增业务逻辑。

### 2.18 统计检验

【位置】：`src/utils/stats.py` -> `run_statistical_tests` [样本外统计检验与多重比较校正]。

【执行流描述】：

前置条件：

- 输入为同一日期交集上的 paired out-of-sample daily net returns。
- `benchmark_returns` 与 `model_returns` 均来自 `daily_returns.csv.net_return`。
- 不允许前向填充缺失收益。

核心步骤：

1. 对每个模型与每个 benchmark 执行日期内连接，生成 `paired_returns`。
2. 计算 `diff_t = model_net_return_t - benchmark_net_return_t`。
3. 执行 block bootstrap：
   1. `block_length` 从配置读取，默认按 `sqrt(n)` 取整。
   2. 重采样次数 `n_bootstrap` 从配置读取。
   3. 输出 mean return diff、Sharpe diff、max drawdown diff 的 confidence interval。
4. 执行 Newey-West / HAC return-diff test：
   1. `lags` 从配置读取；缺省使用 `floor(4*(n/100)^(2/9))`。
   2. 对 `diff_t` 均值执行 HAC 标准误估计。
   3. 输出 t_stat 与 raw p-value。
5. 执行 PSR：
   1. 使用模型样本外日收益计算 Sharpe、skewness、kurtosis。
   2. benchmark Sharpe 为配置值或对应 benchmark Sharpe。
   3. 输出 `psr_value` 与 raw p-value。
6. 执行 DSR：
   1. 读取参与比较的策略数量 `n_strategies`。
   2. 对 PSR 做多重试验调整。
   3. 输出 `dsr_value` 与 raw p-value。
7. 执行 White Reality Check：
   1. 以所有候选模型相对 benchmark 的 `diff_t` 矩阵为输入。
   2. 使用 stationary bootstrap 生成最大绩效统计量分布。
   3. 输出每个模型的 raw p-value。
8. 执行 Hansen SPA：
   1. 对表现差的候选策略做截尾调整。
   2. 使用同一 bootstrap 配置估计 SPA p-value。
9. 执行 Diebold-Mariano：
   1. 只对 auxiliary prediction task 的 forecast error 序列执行。
   2. loss function 默认 squared error，可配置 absolute error。
   3. horizon 等于对应 auxiliary target horizon。
10. 收集所有 raw p-value，按模型族和 benchmark 组执行 Holm-Bonferroni 校正。
11. 写入 `metrics/statistics_summary.csv`。

分支判断：

- 如果 paired 样本长度 `< stats.min_paired_samples`，对应检验 `status="skipped"`，写入 `skip_reason="insufficient_samples"`。
- 如果某模型没有 auxiliary prediction 输出，DM test `status="not_applicable"`。
- 如果 bootstrap 失败，保留点估计，p-value 为空，并记录 `status="failed"` 与 `fail_reason`。
- Holm-Bonferroni 只调整同一比较族内的有效 raw p-value，不调整 skipped/not_applicable 项。

【关键变量说明】：

- `paired_returns [配对样本外收益]`：
  - 类型/范围：`pd.DataFrame`。
  - 取值说明：字段包含 `date`、`model_return`、`benchmark_return`、`diff`。
  - 备注：只使用 out-of-sample。
- `statistics_summary [统计检验输出]`：
  - 类型/范围：CSV。
  - 取值说明：固定字段至少包含 `model_name`、`benchmark_name`、`test_name`、`metric_name`、`effect_size`、`ci_lower`、`ci_upper`、`test_statistic`、`raw_p_value`、`adjusted_p_value`、`adjustment_method`、`significant`、`status`、`skip_reason`。
  - 备注：`adjustment_method` 固定为 `holm_bonferroni` 或空。

`wide_vol_path [成交量矩阵]`：

- 类型/必填性：String / 必填。
- 校验规则：同标准 wide schema。

`wide_turnover_rate_path [换手率矩阵]`：

- 类型/必填性：String / 必填。
- 校验规则：文件必须存在；全空时按 `turnover_rate_required` 决定 drop 相关特征或 fail fast。

`all_metrics_features_path [MetricsFactory 全量特征文件]`：

- 类型/必填性：String / 默认必填。
- 校验规则：parquet；必须包含 `date`、`ts_code`；无重复 `(date, ts_code)`。

`execution_model.execution_price [执行价格模式]`：

- 类型/必填性：String / 必填。
- 校验规则：枚举 `next_open`、`next_close`；默认 `next_open`。

`execution_model.strict_no_lookahead_execution [严格无前视执行]`：

- 类型/必填性：Boolean / 必填。
- 校验规则：默认 true；true 时 observation、gate input、actor/critic 输入不得包含 execution_date 后字段。

`execution_model.signal_timestamp [信号时间戳]`：

- 类型/必填性：String / 必填。
- 校验规则：默认 `close_t_after_market`；主实验只允许 date t close 后、下一执行价成交前形成 `state_t` 与 `action_t`。

`execution_model.delayed_action_execution [延迟动作执行开关]`：

- 类型/必填性：Boolean / 必填。
- 校验规则：当 execution_price 为 `next_close` 时必须 true；默认 false。

`execution_model.same_close_idealized_execution_enabled [同收盘价理想成交开关]`：

- 类型/必填性：Boolean / 必填。
- 校验规则：默认 false；主实验不得启用；启用时 manifest 必须标记。

`execution_model.idealized_execution [理想成交标记]`：

- 类型/必填性：Boolean / 必填。
- 校验规则：默认 false；当 `same_close_idealized_execution_enabled=true` 时必须为 true 或由 ConfigLoader 派生为 true。

`execution_model.t_plus_one [T+1 交易开关]`：

- 类型/必填性：Boolean / 必填。
- 校验规则：默认 false；true 时必须维护 `last_buy_date_per_asset`、`sellable_mask`、`frozen_weight`。

`execution_model.t_plus_one_position_tracking [T+1 持仓跟踪方式]`：

- 类型/必填性：String / 必填。
- 校验规则：枚举 `disabled`、`sellable_mask`、`lot_level`；默认 `disabled`。当 `t_plus_one=false` 时必须为 `disabled`；当 `t_plus_one=true` 时必须为 `sellable_mask` 或 `lot_level`。

`execution_model.cash_enabled [现金开关]`：

- 类型/必填性：Boolean / 必填。
- 校验规则：默认 false；false 时无可用资产必须 fail fast。

`execution_model.partial_fill_enabled [部分成交开关]`：

- 类型/必填性：Boolean / 必填。
- 校验规则：默认 false；false 时不得随机部分成交。

`execution_model.suspend_policy [停牌处理策略]`：

- 类型/必填性：String / 必填。
- 校验规则：枚举 `freeze_and_force_exit_when_available`；默认 `freeze_and_force_exit_when_available`。资产不可用时不得新建仓；已有持仓在无法交易日冻结，待重新可交易后按约束强制退出或重分配。

`execution_model.initial_build_cost [首日建仓成本开关]`：

- 类型/必填性：Boolean / 必填。
- 校验规则：默认 true；false 仅免除首日建仓成本。

`execution_model.fixed_cost_unit [固定成本单位]`：

- 类型/必填性：String / 必填。
- 校验规则：枚举 `nav_fraction`、`currency`；默认 `nav_fraction`。

`cost_model.mode [成本模型模式]`：

- 类型/必填性：String / 必填。
- 校验规则：枚举 `empirical_default`、`calibrated`；默认 `empirical_default`。

`cost_model.market_impact_enabled [市场冲击成本开关]`：

- 类型/必填性：Boolean / 必填。
- 校验规则：默认 true；true 时 `portfolio.initial_capital_currency` 必须大于 0。

`cost_model.market_impact_coef [市场冲击系数]`：

- 类型/必填性：Float / 必填。
- 校验规则：`>= 0`；默认 0.10。

`cost_model.calibration.min_bucket_samples [校准桶最小样本数]`：

- 类型/必填性：Integer / `mode=calibrated` 时必填。
- 校验规则：`> 0`；样本不足必须回退 `empirical_default` 并记录 warning。

`feature_audit.warning_policy [特征 warning 处理策略]`：

- 类型/必填性：String / 必填。
- 校验规则：枚举 `keep`、`drop`、`report_only`；默认 `keep`。

`constraints.partial_rebalance_post_check_policy [部分调仓后约束策略]`：

- 类型/必填性：String / 必填。
- 校验规则：枚举 `report_only`、`project_executed`、`force_full_rebalance`；默认 `report_only`。

`portfolio.initial_nav [初始单位净值]`：

- 类型/必填性：Float / 必填。
- 校验规则：`> 0`，默认 1.0。

`portfolio.initial_capital_currency [初始资金金额]`：

- 类型/必填性：Float / market impact 启用时必填。
- 校验规则：`> 0`，默认 100000000.0。

`portfolio.currency [组合币种]`：

- 类型/必填性：String / 必填。
- 校验规则：默认 `CNY`。

`action.weights [动作权重]`：

- 类型/必填性：array / 环境 step 必填。
- 校验规则：shape `[n_assets]`；finite；会经 mask 和约束修正。

`action.rebalance [是否调仓]`：

- 类型/必填性：Integer / 环境 step 必填。
- 校验规则：0 或 1。

`action.rebalance_intensity [调仓强度]`：

- 类型/必填性：Float / 可选。
- 校验规则：`0 <= value <= 1`；缺失时根据 rebalance 默认 0 或 1。

### 3.3 异常返回

`ERR_CONFIG_UNKNOWN_KEY [未知配置键]`：

- 触发场景：YAML 顶层或 nested key 不在配置 schema 中。
- 处理：fail fast；错误消息包含 key path。

`ERR_CONFIG_INVALID_EXECUTION_MODEL [执行模型配置非法]`：

- 触发场景：`execution_price == "next_close"` 且 `delayed_action_execution == false`。
- 处理：fail fast。

`ERR_DATA_MISSING_FILE [数据文件缺失]`：

- 触发场景：loader 所需 parquet、csv、manifest 不存在。
- 处理：fail fast；列出缺失文件。

`ERR_DATA_NO_AVAILABLE_ASSET [无可用资产]`：

- 触发场景：`asset_universe.csv` 中没有 `status == "ok"`。
- 处理：fail fast。

`ERR_DATA_SCHEMA_MISMATCH [数据 schema 不匹配]`：

- 触发场景：字段缺失、wide columns 不匹配、日期不递增、重复键。
- 处理：fail fast。

`ERR_DATA_TURNOVER_RATE_REQUIRED [换手率字段强制但缺失]`：

- 触发场景：`turnover_rate_required=true` 且 turnover_rate 全空。
- 处理：fail fast。

`ERR_SPLIT_EMPTY [切分后无样本]`：

- 触发场景：window_size、execution horizon、purge 或 embargo 后 split 无可决策日。
- 处理：fail fast。

`ERR_ACTION_NON_FINITE [动作含 NaN/Inf]`：

- 触发场景：环境 step 输入 action weights 非有限。
- 处理：fail fast。

`ERR_CONSTRAINT_REFERENCE_WEIGHTS_REQUIRED [缺少换手约束参考权重]`：

- 触发场景：配置 turnover_limit 但 `reference_weights is None`。
- 处理：fail fast。

`ERR_COST_PORTFOLIO_VALUE_REQUIRED [市场冲击缺少真实资金规模]`：

- 触发场景：market impact 启用但 `portfolio_state.portfolio_value` 缺失。
- 处理：fail fast。

`ERR_EXECUTION_DATE_OUT_OF_RANGE [执行日期越界]`：

- 触发场景：decision date 无法找到 execution date 或 next valuation date。
- 处理：fail fast 或 truncated split，不得伪装 terminated。

`ERR_EXECUTION_INVALID_NAV [净值非法]`：

- 触发场景：`nav_execution <= 0`、`nav_after_cost <= 0` 或 `nav_next <= 0`。
- 处理：fail fast；`nav_next == 0` 必须报错，因为对数收益不可定义。

`ERR_EXECUTION_RETURN_MISSING [执行收益缺失]`：

- 触发场景：已有持仓且资产被判定可交易，但 execution return 或 holding return 缺失。
- 处理：fail fast；只有不可交易冻结资产允许按 0 收益填充并记录。

`ERR_DEVICE_UNAVAILABLE [设备不可用]`：

- 触发场景：用户强制 cuda/mps 但不可用。
- 处理：fail fast。

### 3.4 状态变更

本项目没有数据库状态变更，状态变更发生在内存对象与文件产物。

`PortfolioState.nav [组合单位净值]`：

- 旧状态：step 开始时 `nav_t`。
- 新状态：`nav_next`。
- 触发条件：`PortfolioExecutionCore.execute_step` 成功完成。

`PortfolioState.portfolio_value [组合真实资金规模]`：

- 旧状态：step 开始时真实资金规模。
- 新状态：旧值乘以 `nav_next / nav_t`。
- 触发条件：`PortfolioExecutionCore.execute_step` 成功完成。

`PortfolioState.current_weights [当前权重]`：

- 旧状态：decision date close 后策略可见权重。
- 新状态：执行后权重经过 holding return 漂移后的下一期权重。
- 触发条件：`PortfolioExecutionCore.execute_step` 成功完成。

`PortfolioState.drifted_weights [漂移权重]`：

- 旧状态：上一 step 的漂移权重。
- 新状态：本 step 的 `pre_execution_drifted_weights` 或 holding return 后漂移权重。
- 触发条件：`PortfolioExecutionCore.execute_step` 成功完成。

`PortfolioState.previous_executed_weights [上一期成交权重]`：

- 旧状态：上一 step 的成交权重。
- 新状态：本 step 的 `executed_weights`。
- 触发条件：执行成交成功。

`PortfolioState.running_max_nav [滚动最高净值]`：

- 旧状态：截至上一步的最高 `nav`。
- 新状态：`max(old_running_max_nav, nav_next)`。
- 触发条件：`nav_next` 计算完成。

`PortfolioState.current_drawdown_abs [当前绝对回撤]`：

- 旧状态：上一步回撤。
- 新状态：`max(0, 1 - nav_next / running_max_nav)`。
- 触发条件：`running_max_nav` 更新后。

`PortfolioState.rolling_returns [滚动收益队列]`：

- 旧状态：截至上一步的收益队列。
- 新状态：追加 `net_return`，超过窗口长度时弹出最旧值。
- 触发条件：每个 step 成功完成。

`PortfolioState.step_index [步编号]`：

- 旧状态：当前整数编号。
- 新状态：`old_step_index + 1`。
- 触发条件：每个 step 成功完成。

`PortfolioState.last_buy_date_per_asset [各资产最近买入日]`：

- 旧状态：上一 step 记录。
- 新状态：对 `executed_weights_i > pre_execution_drifted_weights_i + weight_sum_eps` 的资产写入本次 `execution_date`。
- 触发条件：`t_plus_one == true` 且有买入。

`PortfolioState.sellable_mask [T+1 可卖掩码]`：

- 旧状态：上一 decision date 可卖状态。
- 新状态：若 `execution_date > last_buy_date_per_asset_i` 则 true，否则 false。
- 触发条件：每次构造 execution 约束前。

`PortfolioState.frozen_weight [T+1 冻结权重]`：

- 旧状态：上一 step 冻结权重。
- 新状态：对不可卖资产记录不得降低的最低权重；其他资产为 0。
- 触发条件：`t_plus_one == true` 且存在当日不可卖资产。

`run_registry.sqlite:runs.status [运行状态]`：

- 旧状态：`running`。
- 新状态：`success` 或 `failed`。
- 触发条件：实验完成或捕获不可恢复异常。

`results/<run_name>/logs/run_manifest.json [运行清单]`：

- 旧状态：不存在。
- 新状态：写入完整 manifest。
- 触发条件：实验结束前的 artifact 写入阶段。

## 4. 异常处理与边界场景 (Exception Handling)

### 4.1 业务异常清单

场景一 [资产池文件缺失]：

- 对应代码：`src/data/loader.py::load_market_dataset`。
- 触发：`data/processed/asset_universe.csv` 不存在。
- 处理方案：抛出 `ERR_DATA_MISSING_FILE`，消息包含 `{ "kind": "path", "ref": "data/processed/asset_universe.csv" }`。
- 不执行：不得联网自动补数。

场景二 [资产池没有可用资产]：

- 对应代码：`load_market_dataset`。
- 触发：`asset_universe.status == "ok"` 过滤后为空。
- 处理方案：抛出 `ERR_DATA_NO_AVAILABLE_ASSET`。
- 不执行：不得退回旧脚本 `CONFIG['fund_list']`。

场景三 [wide table 列顺序不一致]：

- 对应代码：`validate_wide_table`。
- 触发：读取 wide parquet 后无法 reindex 到 `canonical_asset_order`，或存在缺失资产列。
- 处理方案：抛出 `ERR_DATA_SCHEMA_MISMATCH`。
- 备注：允许列重排，最终必须与标准顺序一致。

场景四 [turnover_rate 全空]：

- 对应代码：`FeatureMatrixBuilder`。
- 触发：`wide_turnover_rate` 对所有 selected assets 全空。
- 处理方案：默认不 fail fast，相关特征进入 provenance，`leakage_check_status="dropped"`，`drop_reason="all_missing"`。
- 例外：`turnover_rate_required=true` 时抛出 `ERR_DATA_TURNOVER_RATE_REQUIRED`。

场景五 [执行模式非法]：

- 对应代码：`ConfigLoader.validate_execution_model`。
- 触发：`execution_price="next_close"` 且 `delayed_action_execution=false`。
- 处理方案：抛出 `ERR_CONFIG_INVALID_EXECUTION_MODEL`。

场景六 [动作权重非法]：

- 对应代码：`PortfolioRebalanceEnv.step`。
- 触发：action weights 含 NaN、Inf、shape 不等于 `[n_assets]`。
- 处理方案：抛出 `ERR_ACTION_NON_FINITE` 或 `ERR_ACTION_SHAPE_MISMATCH`。

场景七 [无可交易资产]：

- 对应代码：`ConstraintManager.project`、`PortfolioExecutionCore.execute_step`。
- 触发：execution date 所有资产不可交易，且 cash disabled。
- 处理方案：抛出 `ERR_CONSTRAINT_NO_AVAILABLE_ASSET` 或按配置进入 cash。
- 备注：不得构造假权重继续运行。

场景八 [市场冲击成本缺少真实资金规模]：

- 对应代码：`CostModel.estimate`。
- 触发：`market_impact_enabled=true` 且 `portfolio_state.portfolio_value is None`。
- 处理方案：抛出 `ERR_COST_PORTFOLIO_VALUE_REQUIRED`。

场景九 [PCA 在 validation/test fit]：

- 对应代码：`assert_pca_not_fit_on_validation_or_test`。
- 触发：`FeatureReductionPipeline.fit` 接收到非 train split 样本。
- 处理方案：抛出 `ERR_LEAKAGE_PCA_FIT_SCOPE`。

场景十 [execution-only 字段进入 observation]：

- 对应代码：`assert_no_execution_field_in_observation`。
- 触发：`market_image`、`gate_input`、`feature_window` 中包含 `holding_simple_return`、`return_from_decision_to_execution`、execution_date 后 amount / volume / availability。
- 处理方案：抛出 `ERR_LEAKAGE_EXECUTION_FIELD`。

### 4.2 边界情况

输入为 `0`：

- `turnover == 0`：`CostModel.estimate` 返回全 0 成本。
- `rebalance_intensity == 0`：执行权重等于 `pre_execution_drifted_weights`，actor loss 权重为 0。
- `fixed_cost == 0`：固定成本项为 0，不影响其他成本。
- `available_asset_count == 0`：不能归一化；cash disabled 时 fail fast。

输入为 `Null`：

- `metrics_features is None`：只在 metrics_factory disabled 时允许；否则 `ERR_DATA_MISSING_FILE`。
- `reference_weights is None`：没有 turnover_limit 时允许；有 turnover_limit 时 fail fast。
- `portfolio_value is None`：market impact disabled 时允许；enabled 时 fail fast。
- `risk_state is None`：risk_state disabled 时允许；启用时必须提供 shape `[n_risk_features]`。

极端数值：

- `alpha` NaN / Inf：`PPOActor` fail fast。
- `ADV20` 过小：用 `adv_eps` 下限。
- `volatility_20d` 缺失：用 `volatility_eps` 并记录 warning。
- `nav_next <= 0`：抛出 `ERR_EXECUTION_INVALID_NAV`。

### 4.3 系统稳定性

文件 IO：

- 所有写文件先写入临时文件，再原子替换目标文件。
- 所有结果文件路径必须位于 `results/<run_name>/` 下。
- 禁止覆盖 `data/processed/`、`data/metrics_factory/`、`data/reports/`，除非显式运行数据生成脚本；正式实验入口不得写这些目录。

网络波动：

- 正式训练、验证、测试、回测入口强制 `offline_mode=true`。
- loader 缺数据时不得联网。
- 网络只存在于 `scripts/rebuild_etf_lof_data.py::fetch_tencent`，正式平台不调用该函数。

设备异常：

- `device.mode=auto`：CUDA 优先，其次 MPS，最后 CPU。
- `device.mode=cuda` 或 `mps`：不可用时 fail fast。
- AMP 只允许 CUDA；CPU/MPS 不启用 AMP。

训练稳定性：

- 每次 optimizer step 前检测 loss NaN / Inf。
- 梯度裁剪使用 `max_grad_norm`。
- checkpoint 保存 best 与 last。
- 发生训练异常时，`run_registry.sqlite` 写入 failed，保留已有日志。

## 6. 非功能性约束 (Constraints)

### 6.1 安全约束

`permission_check [权限校验]`：

- 实现位置：`src/config.py::assert_path_allowed`。
- 校验逻辑：
  1. 对所有输入路径执行 `Path.resolve()`。
  2. 校验 resolved path 是否位于 `security.path_whitelist` 任一路径下。
  3. 不满足则抛出 `ERR_SECURITY_PATH_DENIED`。
- 适用范围：config、data、output、checkpoint load、optional benchmark CSV。

`offline_mode [离线模式]`：

- 实现位置：`src/config.py` 与 `src/data/loader.py`。
- 默认值：true。
- 行为：正式平台不得发起网络请求；缺数据时 fail fast。

`safe_torch_load [安全 checkpoint 加载]`：

- 实现位置：`src/utils/checkpoint.py::load_checkpoint`。
- 行为：优先使用安全权重加载；不加载未受信任 pickle。

### 6.2 性能约束

数据读取：

- wide parquet 读取后按资产顺序缓存到 `MarketDatasetBundle.wide`。
- 不重复读取同一 parquet；不同模块通过 bundle 共享只读引用。

MarketImageDataset：

- 默认 lazy / on-demand materialization。
- `materialize_market_images=false` 时，`market_images` 可为 None 或轻量索引对象。
- `__len__` 与 `__getitem__` 必须稳定。
- 小样本测试可显式调用 `materialize()`。

特征处理：

- PCA 输入样本为 train split 的 `(date, asset)` 行。
- availability_mask 不进入 PCA，避免无意义维度和泄漏风险。
- MetricsFactory 全量 3048 特征先审计，再由配置决定是否入模。

训练：

- batch 移动通过 `move_batch_to_device`。
- CPU/MPS 禁用 AMP。
- CUDA AMP 可选。
- checkpoint 写入不阻塞主训练循环太久；至少 best 和 last 保存。
