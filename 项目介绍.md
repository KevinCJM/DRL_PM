# 项目介绍

## 1. 项目目的

本项目是一个面向论文实验的深度强化学习资产配置平台，目标是在统一数据、统一执行、统一成本、统一约束和统一评估口径下，对比传统组合策略、原生深度强化学习基准、相关论文复现模型，以及后续提出的新模型。

当前主实验资产池已经从旧 ETF/LOF 池切换到 Core-13 v2。旧 17 资产结果、旧 Core-13 早期结果、旧 HPO/checkpoint/paper tables 不应继续作为正式论文结果复用。

核心研究问题：

```text
在中国 ETF/LOF Core-13 资产池上，深度强化学习模型是否能在累计收益、换手率、交易成本和风险之间形成优于传统策略与强 RL baseline 的可复核优势。
```

## 8. 当前最大风险：低换手/买入持有退化

多轮实验显示，多个 RL 或新模型出现超低换手率，行为接近 buy-and-hold。该现象可能来自：

```text
monthly scheduler 限制交易机会；
策略层提前用 scheduler mask 清零 raw intent；
rho / gate scoring 中收益、换手、成本量纲不一致；
HPO objective 过度奖励低成本低换手；
P16 learned rho 训练预算不足或 rho 分布坍缩；
engine/backtest 与 training env raw/final action 语义不一致。
```

当前已形成详细改造文档：

```text
{ "kind": "file", "ref": "换手优化_详细设计.md" }
```

该文档的关键冻结方向：

```text
策略只输出 raw_rho / raw intent。
BacktestEngine 与 PortfolioRebalanceEnv 统一生成 final_rho / execution_action。
weekly_gate 必须由 scheduler 限制执行机会。
daily_gate_with_cost_constraint 才允许 scheduler 不阻断模型 gate。
evaluate_pre_post_no_mutation() 不得修改 scheduler 状态。
commit_scheduler_decision() 是唯一 scheduler 状态提交入口。
expected_alpha_horizon 必须由 decision-visible 输入计算，不得泄漏 future return。
```

如果 CAGE-EIIE 继续低换手但收益最高，论文不能直接声称 learned active allocation。更稳妥的主张是：

```text
CAGE-EIIE learns a conservative execution overlay over EIIE allocation candidates and improves the return-cost trade-off under the Core-13 v2 protocol.
```

如果 active protocol 下收益下降，应如实报告：

```text
原高收益来自低活跃/近 buy-and-hold execution discipline，而不是频繁主动交易能力。
```
